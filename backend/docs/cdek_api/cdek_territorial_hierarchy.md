# Иерархия территориального деления России в API CDEK

## Введение

API CDEK (v2.0) использует **гибридную модель** адресации, сочетающую:

1. **Собственные справочники СДЭК** — числовые коды населённых пунктов (`code`), UUID (`city_uuid`), коды регионов (`region_code`)
2. **Государственные классификаторы** — ФИАС (`fias_guid`), КЛАДР (`kladr_code`, deprecated), ISO 3166-1 (`country_code`)
3. **Плоская строка адреса** (`address`) — для уровней ниже населённого пункта (улица, дом, квартира)

Структурированная иерархия в CDEK доходит только до **населённого пункта**. Всё, что ниже (улица, дом, квартира), передаётся единой строкой `address`.

---

## Государственная иерархия vs. API CDEK

```
Государственная иерархия (ОКТМО + ФИАС/ГАР)          API CDEK
─────────────────────────────────────────────          ────────
Страна (ISO 3166-1: RU)                               → country + country_code (ISO alpha-2)
  │
  ├─ Субъект Федерации (89 шт.)                        → region + region_code     ← Уровень 1
  │    область, край, республика, город фед.              + fias_region_guid (deprecated)
  │    значения, автономный округ                         + kladr_region_code
  │
  ├─ Муниц. район / Городской округ (~2 340 шт.)      → sub_region               ← Уровень 2
  │    ОКТМО: цифра 3 (P1) = тип
  │
  ├─ Поселение (городское / сельское)                  ─ ОТСУТСТВУЕТ ─
  │    ОКТМО: цифра 6 (P2)
  │
  ├─ Населённый пункт (~155 000 шт.)                  → city + code + city_uuid  ← Уровень 3
  │    город, ПГТ, село, деревня                         + fias_guid + kladr_code (deprecated)
  │
  ├─ Элемент планировочной структуры                   ┐
  │    микрорайон, СНТ (ГАР: уровень 7)                │
  ├─ Улично-дорожная сеть                              │
  │    улица, проспект (ГАР: уровень 8)                ├→ address (единая строка) ← Уровень 4
  ├─ Здание / сооружение                               │
  │    дом, корпус, строение (ГАР: уровень 10)         │
  ├─ Помещение                                         │
  │    квартира, офис (ГАР: уровень 11)                ┘
  │
  ├─ Земельный участок (ГАР: уровень 9)               ─ ОТСУТСТВУЕТ ─
  ├─ Комната в помещении (ГАР: уровень 12)            ─ ОТСУТСТВУЕТ ─
  └─ Машино-место (ГАР: уровень 17)                   ─ ОТСУТСТВУЕТ ─

Дополнительно:
  • Координаты: latitude + longitude (центр населённого пункта)
  • Почтовый индекс: postal_code
  • Часовой пояс: time_zone
  • Лимит наложенного платежа: payment_limit
  • Код ПВЗ: буквенно-цифровой код (например, "MSK93", "NSK1")
```

**Ключевые особенности:**

- Структурированные поля до уровня города, ниже — единая строка `address`
- **Тройная идентификация** населённого пункта: `code` (СДЭК) + `city_uuid` (СДЭК) + `fias_guid` (ФИАС)
- **Двойная идентификация** региона: `region_code` (СДЭК) + `kladr_region_code` (КЛАДР)
- КЛАДР-поля помечены как **deprecated** — переход на ФИАС

## 4 уровня адресной иерархии в API CDEK

### Уровень 0: Страна (`country` + `country_code`)

| Поле           | Тип       | Описание               | Пример     |
| -------------- | --------- | ---------------------- | ---------- |
| `country_code` | string(2) | Код ISO 3166-1 alpha-2 | `"RU"`     |
| `country`      | string    | Название страны        | `"Россия"` |

API CDEK поддерживает международную доставку. `country_code` — стандартный ISO-код, по умолчанию `"RU"`.

### Уровень 1: Регион (`region` + `region_code`)

Субъект Российской Федерации.

| Поле                | Тип    | Описание                                  | Пример                 |
| ------------------- | ------ | ----------------------------------------- | ---------------------- |
| `region`            | string | Название региона                          | `"Новосибирская обл."` |
| `region_code`       | int32  | Код региона СДЭК (собственный справочник) | `23`                   |
| `fias_region_guid`  | uuid   | ФИАС GUID региона (**deprecated**)        | `"..."`                |
| `kladr_region_code` | string | Код КЛАДР региона                         | `"54"`                 |

**Получение списка регионов:** `GET /v2/location/regions`

Параметры фильтрации:

- `country_codes` — массив ISO alpha-2 кодов стран
- `fias_region_guid` — GUID ФИАС (deprecated)
- `size`, `page` — пагинация
- `lang` — локализация (`rus`, `eng`, `zho`)

### Уровень 2: Район региона (`sub_region`)

Муниципальный район или городской округ внутри субъекта РФ.

| Поле         | Тип    | Описание                | Пример                |
| ------------ | ------ | ----------------------- | --------------------- |
| `sub_region` | string | Название района региона | `"Одинцовский район"` |

- Необязательное поле
- Нет собственного кода СДЭК — только строковое название
- Может быть `null` для городских округов

### Уровень 3: Населённый пункт (`city` + `code` + `city_uuid`)

Город, посёлок, село, деревня — ключевой уровень маршрутизации в CDEK.

| Поле            | Тип    | Описание                     | Пример                                   |
| --------------- | ------ | ---------------------------- | ---------------------------------------- |
| `code`          | int32  | Код населённого пункта СДЭК  | `270` (Новосибирск)                      |
| `city_uuid`     | uuid   | UUID в ИС СДЭК               | `"..."`                                  |
| `city`          | string | Название населённого пункта  | `"Новосибирск"`                          |
| `fias_guid`     | uuid   | ФИАС GUID населённого пункта | `"8dea00e3-9aab-4d8e-887c-ef2aaa546456"` |
| `kladr_code`    | string | Код КЛАДР (**deprecated**)   | `"5400000100000"`                        |
| `longitude`     | double | Долгота центра города        | `82.934`                                 |
| `latitude`      | double | Широта центра города         | `55.028`                                 |
| `time_zone`     | string | Часовой пояс                 | `"Asia/Novosibirsk"`                     |
| `payment_limit` | double | Лимит наложенного платежа    | `-1` (нет лимита)                        |

**Получение населённых пунктов:** `GET /v2/location/cities`

Параметры фильтрации:

| Параметр            | Описание                             |
| ------------------- | ------------------------------------ |
| `country_codes`     | Массив ISO alpha-2 кодов стран       |
| `region_code`       | Код региона СДЭК                     |
| `kladr_region_code` | Код КЛАДР региона                    |
| `fias_region_guid`  | GUID ФИАС региона (deprecated)       |
| `kladr_code`        | Код КЛАДР населённого пункта         |
| `fias_guid`         | GUID ФИАС населённого пункта         |
| `postal_code`       | Почтовый индекс                      |
| `code`              | Код населённого пункта СДЭК          |
| `city`              | Название (полное совпадение)         |
| `payment_limit`     | Фильтр по лимиту наложенного платежа |
| `size`, `page`      | Пагинация                            |
| `lang`              | Локализация                          |

**Дополнительные методы определения города:**

| Эндпоинт                          | Описание               | Входные данные                   |
| --------------------------------- | ---------------------- | -------------------------------- |
| `GET /v2/location/suggest/cities` | Подсказки по названию  | `name` (строка) + `country_code` |
| `GET /v2/location/coordinates`    | Город по координатам   | `latitude` + `longitude`         |
| `GET /v2/location/postalcodes`    | Индексы по коду города | `city_code` (int)                |

### Уровень 4: Адрес (`address`) — плоская строка

Всё, что ниже населённого пункта, передаётся единой строкой.

| Поле          | Тип         | Описание        | Пример              |
| ------------- | ----------- | --------------- | ------------------- |
| `address`     | string(255) | Строка адреса   | `"ул. Блюхера, 32"` |
| `postal_code` | string      | Почтовый индекс | `"630009"`          |

> **CDEK не разбирает адрес на компоненты.** В отличие от Почты России (16+ полей) и Yandex (11 полей), CDEK использует единую строку `address` для всего, что ниже города: улица, дом, корпус, строение, квартира — всё в одном поле.

**Поле `address` обязательно** в моделях `RequestFromLocationDto` и `RequestToLocationDto` при создании заказа с тарифом «от/до двери».

## Модели локации в API

CDEK использует несколько вариантов модели локации для разных операций:

### Модели для создания заказа

| Модель                   | Использование     | Обязательные поля |
| ------------------------ | ----------------- | ----------------- |
| `RequestFromLocationDto` | Адрес отправления | `address`         |
| `RequestToLocationDto`   | Адрес получения   | `address`         |

Полный набор полей (оба идентичны):

```
RequestFromLocationDto / RequestToLocationDto
├── code              — код нас. пункта СДЭК (int32)
├── city_uuid         — UUID в ИС СДЭК
├── city              — название нас. пункта
├── fias_guid         — GUID ФИАС нас. пункта
├── kladr_code        — код КЛАДР (deprecated)
├── country_code      — ISO alpha-2 (по умолчанию "RU")
├── country           — название страны
├── region            — название региона
├── region_code       — код региона СДЭК (int32)
├── fias_region_guid  — GUID ФИАС региона
├── sub_region        — район региона
├── longitude         — долгота
├── latitude          — широта
├── time_zone         — часовой пояс
├── payment_limit     — лимит наложенного платежа
├── address           — строка адреса (ОБЯЗАТЕЛЬНО)
└── postal_code       — почтовый индекс
```

### Модель для расчёта тарифа

| Модель                  | Поля                                                                                                 |
| ----------------------- | ---------------------------------------------------------------------------------------------------- |
| `CalculatorLocationDto` | `code`, `postal_code`, `country_code`, `city`, `address`, `contragent_type`, `longitude`, `latitude` |

> Упрощённая модель — только ключевые поля для определения маршрута.

### Модель для ПВЗ/постаматов

| Модель              | Использование    | Обязательные поля                                                                                                |
| ------------------- | ---------------- | ---------------------------------------------------------------------------------------------------------------- |
| `OfficeLocationDto` | Адрес офиса СДЭК | `address`, `address_full`, `city`, `city_code`, `country_code`, `latitude`, `longitude`, `region`, `region_code` |

```
OfficeLocationDto
├── country_code   — ISO alpha-2 (обязательно)
├── region_code    — код региона СДЭК (обязательно)
├── region         — название региона (обязательно)
├── city_code      — код нас. пункта СДЭК (обязательно)
├── city           — название нас. пункта (обязательно)
├── city_uuid      — UUID в ИС СДЭК
├── fias_guid      — GUID ФИАС
├── postal_code    — почтовый индекс
├── longitude      — долгота (обязательно)
├── latitude       — широта (обязательно)
├── address        — строка адреса (обязательно)
└── address_full   — полный адрес (страна, регион, город...) (обязательно)
```

### Модель ответа заказа

| Модель                    | Использование              |
| ------------------------- | -------------------------- |
| `ResponseFromLocationDto` | Адрес отправления в ответе |
| `ResponseToLocationDto`   | Адрес получения в ответе   |

Обязательные поля: `code`, `city`, `country_code`. Полный набор аналогичен `RequestFromLocationDto`.

## Система идентификаторов

CDEK использует **многослойную систему идентификаторов**, соединяющую собственные справочники с государственными:

### Идентификаторы населённого пункта

| Идентификатор | Система    | Тип    | Уникальность | Статус         |
| ------------- | ---------- | ------ | ------------ | -------------- |
| `code`        | СДЭК       | int32  | Внутри СДЭК  | **Основной**   |
| `city_uuid`   | СДЭК       | uuid   | Глобальный   | Активный       |
| `fias_guid`   | ФИАС (ФНС) | uuid   | Глобальный   | Активный       |
| `kladr_code`  | КЛАДР      | string | Глобальный   | **Deprecated** |

### Идентификаторы региона

| Идентификатор       | Система | Тип    | Статус         |
| ------------------- | ------- | ------ | -------------- |
| `region_code`       | СДЭК    | int32  | **Основной**   |
| `kladr_region_code` | КЛАДР   | string | Активный       |
| `fias_region_guid`  | ФИАС    | uuid   | **Deprecated** |

### Идентификаторы страны

| Идентификатор  | Система    | Формат  | Пример                 |
| -------------- | ---------- | ------- | ---------------------- |
| `country_code` | ISO 3166-1 | alpha-2 | `"RU"`, `"KZ"`, `"BY"` |

### Идентификаторы ПВЗ/офисов

| Идентификатор | Описание              | Пример              |
| ------------- | --------------------- | ------------------- |
| `code` (ПВЗ)  | Буквенно-цифровой код | `"MSK93"`, `"NSK1"` |

> **ПВЗ идентифицируются строковым кодом** (не UUID и не числовым ID). Формат: `{код города}{номер}` (например, `MSK93` — Москва, ПВЗ #93).

## Связь с государственными классификаторами

| Классификатор          | Как используется в API CDEK                                                                                                                                      |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **ISO 3166-1 alpha-2** | `country_code` — стандартный код страны. Используется во всех моделях                                                                                            |
| **ФИАС/ГАР**           | `fias_guid` — GUID населённого пункта (активный). `fias_region_guid` — GUID региона (**deprecated**). Доступны для поиска городов и офисов                       |
| **КЛАДР**              | `kladr_code` — код города (**deprecated**). `kladr_region_code` — код региона (активный). Переход с КЛАДР на ФИАС в процессе                                     |
| **ОКТМО**              | **Не используется**                                                                                                                                              |
| **Почтовый индекс**    | `postal_code` — поддерживается как для поиска городов, так и для определения адреса. Отдельный эндпоинт `/v2/location/postalcodes` для получения индексов города |

- **CDEK: 17 полей** в `LocationDto`, но адрес ниже города — единая строка

- **Yandex: 11 полей**, адрес частично декомпозирован (6 уровней)
- **Только CDEK** имеет собственные числовые коды для городов и регионов
- **Только CDEK** интегрируется напрямую с ФИАС/КЛАДР через GUID/коды
- **Только CDEK** использует ISO 3166-1 для стран (Yandex и Почта — строковые названия)

## ПВЗ и постаматы как альтернатива адресу

При доставке «до склада» или «до постамата» вместо `to_location` используется `delivery_point` — строковый код ПВЗ.

| Тип офиса | Значение `type` | Описание                    |
| --------- | --------------- | --------------------------- |
| ПВЗ       | `PVZ`           | Пункт выдачи заказов        |
| Постамат  | `POSTAMAT`      | Автоматизированный постамат |
| Все       | `ALL`           | Оба типа                    |

**Получение списка офисов:** `GET /v2/deliverypoints`

Фильтрация:

| Параметр                    | Описание                      |
| --------------------------- | ----------------------------- |
| `code`                      | Код ПВЗ (например, `"NSK1"`)  |
| `type`                      | Тип: `PVZ`, `POSTAMAT`, `ALL` |
| `city_code`                 | Код города СДЭК               |
| `postal_code`               | Почтовый индекс               |
| `country_code`              | ISO alpha-2                   |
| `region_code`               | Код региона СДЭК              |
| `fias_guid`                 | GUID ФИАС города              |
| `have_cashless`             | Терминал оплаты               |
| `have_cash`                 | Приём наличных                |
| `allowed_cod`               | Наложенный платеж             |
| `is_dressing_room`          | Примерочная                   |
| `weight_max` / `weight_min` | Фильтр по весу                |
| `is_handout`                | Пункт выдачи                  |
| `is_reception`              | Приём заказов                 |
| `is_marketplace`            | Маркетплейс                   |
| `is_ltl`                    | Сборный груз (LTL)            |
| `fulfillment`               | Зона фулфилмента              |

> **Взаимоисключение:** `shipment_point`/`delivery_point` (код ПВЗ) и `from_location`/`to_location` (адрес) **не могут использоваться одновременно**. Для тарифов «от склада» обязателен `shipment_point`, для «от двери» — `from_location`.

---

## Особые случаи

| Ситуация                   | Как обрабатывается в API                                                                                                                                       |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Москва, Питер, Севастополь | `region` = `city` (аналогично другим API). `sub_region` может быть `null`                                                                                      |
| Международная доставка     | `country_code` ≠ `"RU"`. Поддерживаются Казахстан, Беларусь, Узбекистан и другие страны                                                                        |
| Населённый пункт без ФИАС  | `fias_guid` = `null`. Город идентифицируется через `code` (СДЭК)                                                                                               |
| Переход с КЛАДР на ФИАС    | `kladr_code`, `fias_region_guid` помечены `deprecated`. Рекомендуется использовать `fias_guid` для городов, `kladr_region_code` или `region_code` для регионов |
| Наложенный платеж          | `payment_limit` в модели города определяет лимит: `-1` = без лимита, `0` = не принимается                                                                      |
| Доставка «от двери»        | Обязателен `from_location.address`. Город определяется через `code`, `fias_guid`, `postal_code` или `city`                                                     |
| Доставка «до склада»       | Обязателен `delivery_point` (код ПВЗ). `to_location` не передаётся                                                                                             |
| Локализация                | Параметр `lang`: `rus` (по умолчанию), `eng`, `zho` — влияет на названия регионов и городов                                                                    |

## API-эндпоинты территориальных справочников

| Эндпоинт                      | Метод | Описание                     | Ключевые параметры                                                      |
| ----------------------------- | ----- | ---------------------------- | ----------------------------------------------------------------------- |
| `/v2/location/regions`        | GET   | Список регионов              | `country_codes`, `fias_region_guid`, `lang`                             |
| `/v2/location/cities`         | GET   | Список населённых пунктов    | `code`, `city`, `region_code`, `fias_guid`, `kladr_code`, `postal_code` |
| `/v2/location/suggest/cities` | GET   | Подсказки по названию города | `name` (обязательно), `country_code`                                    |

| `/v2/location/postalcodes`    | GET   | Почтовые индексы города      | `city_code` (обязательно)                                               |
| `/v2/deliverypoints`          | GET   | Список ПВЗ/постаматов        | `code`, `city_code`, `region_code`, `type`, `fias_guid`                 |
