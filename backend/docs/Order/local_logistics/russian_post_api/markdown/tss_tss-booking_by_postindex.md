# Оформление бронирования

Бронирует отправление на таймслот по id. В качестве ШПИ может использоваться как ШПИ, так и UUID.

Выполняется проверка наличия указанного в запросе таймслота путем вызова метода: timeslotsByAddress/external-timeslots-by-postindex. В случае несовпадения id таймслота в запросе и id таймслота из ответа возвращается ошибка "TIMESLOT_NOT_FOUND".

## Параметры запроса

| Параметр | Значение |
|----------|----------|
| Локальный URL | **/external/v1/booking-by-postindex** |
| Тип | **POST** |

## Заголовки запроса

| Заголовок | Значение |
|-----------|----------|
| **Authorization** | **AccessToken {token}** |
| **X-User-Authorization** | **Basic {key}** |
| **Content-Type** | **application/json;charset=UTF-8** |

## Тело запроса

```json
{
  "uuid": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "timeslotId": 0,
  "contactName": "string",
  "phone": "string",
  "postIndexFrom": "string",
  "postIndexTo": "string",
  "contractNumber": "string",
  "plannedShippingDate": "2023-02-03",
  "address": "string",
  "workTypeCode": "string",
  "mailTypeCode": 0,
  "mailCtgCode": 0
}
```

| Параметр | Тип | Описание |
|----------|-----|----------|
| uuid | Строка | UUID |
| timeslotId | Целое число | Идентификатор временного интервала |
| contactName | Строка (Опционально) | Имя получателя (максимальная длина 200 символов) |
| phone | Строка | Телефон получателя (максимальная длина 50 символов) |
| postIndexFrom | Строка | Почтовый индекс отправителя (6 цифр) |
| postIndexTo | Строка | Почтовый индекс получателя (6 цифр) |
| contractNumber | Строка | Номер договора (максимальная длина 50 символов) |
| plannedShippingDate | Строка | Плановая дата в формате: yyyy-MM-dd |
| address | Строка | Адрес получателя (максимальная длина 255 символов) |
| workTypeCode | Строка | Код типа работ. В текущей реализации может принимать только значение "delivery" (Доставка) |
| mailTypeCode | Целое число | Код типа отправления |
| mailCtgCode | Целое число | Код категории почтового отправления |

## Ответ на запрос (ошибка)

```json
{
  "code": "string",
  "message": "string",
  "errors": [
    {
      "field": "string",
      "message": "string"
    }
  ]
}
```

| Параметр | Тип | Описание |
|----------|-----|----------|
| code | Строка | Текстовый идентификатор ошибки |
| message | Строка | Подробное описание ошибки поля входных данных |
| errors | Массив | Возвращается при наличии ошибок в полях входных данных |
| errors[].field | Строка | Текстовый идентификатор поля входных данных |
| errors[].message | Строка | Подробное описание ошибки поля входных данных |
