# Создание заказов с помощью расширения Chrome браузера

Цель данного раздела:

- проверить авторизацию на API сервисе. Здесь будут указаны реальные токен авторизации и ключ авторизации;
- создание отправлений через **API Онлайн-сервиса "Отправка"**.

Здесь рассматриваются базовые подходы к созданию REST запросов, которые могут быть использованы для отладки других сервисов **API Онлайн-сервиса "Отправка"**.

### 1 шаг: Скачайте и установите плагин

1. Перейдите по ссылке и установите ["Advanced REST client" плагин](https://chrome.google.com/webstore/detail/advanced-rest-client/hgmloofddffdnphfgcellkdfbfbjeloo?hl=en-US).

### 2 шаг: Запустите плагин

1. Откройте новую пустую страницу, вставьте ссылку **chrome://apps** и нажмите "Enter" на клавиатуре
2. Запустите "Advanced REST client" плагин.

### 3 шаг: Создайте новый запрос для добавления заказов

После запуска:

1. Укажите ссылку, состоящую из адреса сервера и локального адреса **/1.0/user/backlog**. Локальный адрес взят из [метода "Создание заказа без ШПИ"](#/orders-creating_order). Ссылки для других методов уточняйте в документации.
Таким образом, ссылка должна выглядеть так:
**{host}/1.0/user/backlog**

2. Укажите тип запроса **PUT**. Данный тип взят из [метода "Создание заказа без ШПИ"](#/orders-creating_order). Типы для других методов уточняйте в документации.
Это могут быть следующие типы: "GET", "POST", "PUT" или "DELETE".

3. Нажмите на закладку "Headers form" и добавьте 3 заголовка:
   - Добавьте параметр авторизации приложения **Authorization** и укажите префикс **AccessToken**, символ "пробела" и **[токен авторизации приложения](#/authorization-token)**;
   - Добавьте параметр авторизации пользователя **X-User-Authorization** и укажите префикс **Basic**, символ "пробела" и **[ключ авторизации пользователя](#/authorization-key)**;
   - Добавьте параметр формата ожидаемого результата **Content-Type** и укажите **application/json;charset=UTF-8**

4. В теле запроса **Raw Payload** укажите один или несколько заказов согласно формату [метода "Создание заказа без ШПИ"](#/orders-creating_order).

Пример:
Мы должны добавить две посылки:
1) "117105, г Москва, ш Варшавское, д 37", 1 кг, заказ в магазине "001", тел.: 79459562067, получатель: Иванов Иван Иванович
2) "630084, обл Новосибирская, г Новосибирск, проезд Газовый, д 13, кв 99", 20 кг, заказ в магазине "002", тел.: 79458712076, получатель: Сидоров Сергей Владимирович

Для этого:
1) Разделите адреса на части вручную или с помощью [метода нормализации адреса](#/nogroup-normalization_adress). Смотрите пример: [Проверка и нормализация адреса с помощью SoapUI](#/usecases-soapui-clean_address).
2) Для каждого адреса мы готовим JSON структуры согласно формату [метода "Создание заказа без ШПИ"](#/orders-creating_order).
3) Эти JSON структуры мы должны объединить в группу. Для обозначения группы используются "квадратные скобки" ( **[** и **]** ). Для разделения элементов внутри группы используется "запятая" ( **,** ).

Таким образом, мы получаем окончательное тело запроса:

```json
[
    {
        "address-type-to": "DEFAULT",
        "given-name": "Иван",
        "house-to": "37",
        "index-to": 117105,
        "mail-category": "ORDINARY",
        "mail-direct": 643,
        "mail-type": "POSTAL_PARCEL",
        "mass": 1000,
        "middle-name": "Иванович",
        "order-num": "001",
        "place-to": "г Москва",
        "region-to": "г Москва",
        "street-to": "ш Варшавское",
        "surname": "Иванов",
        "tel-address": 79459562067,
        "transport-type": "SURFACE"
    },
    {
        "address-type-to": "DEFAULT",
        "given-name": "Сергей",
        "house-to": "13",
        "index-to": 630084,
        "mail-category": "ORDINARY",
        "mail-direct": 643,
        "mail-type": "POSTAL_PARCEL",
        "mass": 20000,
        "middle-name": "Владимирович",
        "order-num": "002",
        "place-to": "г Новосибирск",
        "region-to": "обл Новосибирская",
        "room-to": "99",
        "street-to": "проезд Газовый",
        "surname": "Сидоров",
        "tel-address": 79458712076,
        "transport-type": "SURFACE"
    }
]
```

**Внимание! Если метод групповой, то необходимо создавать группу даже для одного элемента JSON структуры.**

Пример:

```json
[
    {
        "address-type-to": "DEFAULT",
        "given-name": "Сергей",
        "house-to": "13",
        "index-to": 630084,
        "mail-category": "ORDINARY",
        "mail-direct": 643,
        "mail-type": "POSTAL_PARCEL",
        "mass": 200,
        "middle-name": "Владимирович",
        "order-num": "002",
        "place-to": "г Новосибирск",
        "region-to": "обл Новосибирская",
        "room-to": "99",
        "street-to": "проезд Газовый",
        "surname": "Сидоров",
        "tel-address": 79458712076,
        "transport-type": "SURFACE"
    }

```

### 4 шаг: Выполните запрос

1. Нажмите кнопку **SEND**
2. Будут возвращены идентификаторы заданных заказов или ошибки. Можно посмотреть **Raw** или **JSON** форматы.
3. Для успешно созданных заказов будут возвращены идентификаторы.
4. Для несозданных заказов будут возвращены описания ошибок.
В данном случае: **ILLEGAL_MASS_EXCESS** (Превышена допустимая масса). В "details" описана максимальная масса в граммах (10 кг) для заказа в 1-й "позиции" (2-й элемент списка).

**Внимание! Нумерация начинается с нулевого элемента (0, 1, 2 и т.д.).**
