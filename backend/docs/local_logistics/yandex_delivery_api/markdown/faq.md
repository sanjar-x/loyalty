## Общие вопросы

### С какого статуса начинается платная отмена?

Платная отмена начинается со статуса `SORTING_CENTER_AT_START`.

### В какой момент можно отменить заказ?

Заказ можно отменить в любой момент до того, как посылки были переданы курьеру — до перехода в статус `DELIVERY_TRANSPORTATION_RECIPIENT`. Используйте метод [Отмена заявки](yandex_delivery_other_day_request_cancel.md).

### Что такое platform_station_id?

`platform_station_id` — уникальный идентификатор склада в системе Яндекс Доставки. Используется как для вашего склада, так и для ПВЗ и точек самопривоза.

### Как доставить заказ до ПВЗ?

1. Получите `platform_station_id` ПВЗ через метод [Получение списка точек самопривоза и ПВЗ](yandex_delivery_other_day_pickup_points_list.md).
2. Для фильтрации по региону узнайте `geo_id` через метод [Получение geo_id](yandex_delivery_other_day_location_detect.md).
3. Оформите заявку через метод [Создание заказа](yandex_delivery_other_day_offers_create.md), передав `destination.platform_station` с идентификатором ПВЗ.

### Что такое geo_id?

`geo_id` — уникальный идентификатор населенного пункта. В больших городах может быть несколько `geo_id` в зависимости от переданного адреса.

### Что такое places и items?

| Понятие | Описание |
|---------|----------|
| `items` | Товары, которые упакованы в `places` |
| `places` | Коробки (места), которые будут погружены в машину при отгрузке |

### Поддерживаете ли вы маркировку «Честный знак»?

Да, поддерживается версия 1.2 и передача криптохвоста. В поле `marking_code` укажите GTIN (14 символов) и серийный номер (13 символов). Маркировка товаров не печатается на чеках. Ответственность за вывод товаров из оборота остаётся на клиенте.

### Можно ли напечатать свой ярлык? Какие требования?

Да. Требования:

| Требование | Описание |
|------------|----------|
| Штрихкод | Уникальный на каждое грузовое место заказа |
| Содержание | Номер заказа, название магазина, адрес получателя, дата/время доставки, контактные данные |
| Сканирование | Ярлык должен правильно сканироваться и быть стандартных размеров |
| Расположение | На ровной поверхности упаковки, не на рёбрах коробки |
| Целостность | При открывании коробки ярлык должен сохранять целостность |

[Пример ярлыка](https://yastatic.net/s3/doc-binary/src/dev/logistics/ru/delivery-api/files/bar-codes.jpg). Подробнее: [Маркировать товар](https://yandex.com/support/delivery-profile/ru/other-day/mark-rules).

### Как узнать интервалы доставки в другой город?

Используйте метод [Получение интервалов доставки](yandex_delivery_other_day_offersinfo_post.md).

### Какое минимально доступное время забора после оформления заказа?

Если планируется забор сегодня, создайте заказ до обозначенного времени забора.

### Что такое operator_request_id?

`operator_request_id` — уникальный ключ для каждого подтвержденного заказа, задаётся на стороне клиента. При создании нового заказа используйте другой ключ.

### В каком формате передавать адрес?

Пример корректного адреса: `Россия, Москва, улица Пушкина, д. 8`.

| Правило | Описание |
|---------|----------|
| Запятые | Разделяйте разные части адреса запятыми |
| Индексы | Не указывайте почтовые индексы |
| Квартира | Не указывайте в адресе. Используйте поле `destination.custom_location.details.room` |
| Детали | Код домофона, номер подъезда и т.д. — в поле `comment` |

### Как сделать оплату при получении?

Заполните поле `billing_info` при создании заказа.

| Способ оплаты | Описание |
|---------------|----------|
| `card_on_receipt` | Оплата картой при получении (вкл. postpay) |
| `postpay` | Оплата картой в приложении GO или по ссылке из СМС |

### Что такое Postpay и как он работает?

Postpay — оплата картой через приложение GO или по ссылке из СМС.

| Шаг | Действие |
|-----|----------|
| 1 | Получатель выбирает товары в ПВЗ (если `partial_refuse = true`) |
| 2 | Сотрудник ПВЗ подтверждает состав и отправляет уведомление |
| 3 | Получатель нажимает на пуш, открывается страница оплаты |
| 4 | Нажимает «Оплатить» |
| 5 | Выбирает способ: СБП или банковская карта |

Для отправки с postpay укажите `payment_method: postpay` в `billing_info`.

### Как работает страховая стоимость?

Подробнее: [Возмещение ущерба](https://yandex.com/support/delivery-profile/ru/security/insurance-cargo-other-day).

### В каких единицах передавать данные?

| Параметр | Единица измерения |
|----------|-------------------|
| Объём | см³ |
| Габариты | сантиметры |
| Вес | граммы |
| Стоимость | копейки |
| `unit_price` | Стоимость за единицу товара (в копейках) |

### В каком формате передавать номер телефона?

Без знака плюс: `79123456789`.

### Есть ли тестирующая система?

| Параметр | Значение |
|----------|----------|
| Хост | `https://b2b.taxi.tst.yandex.net/` |
| Тестовый токен | `y2_AgAAAAD04omrAAAPeAAAAAACRpC94Qk6Z5rUTgOcTgYFECJllXYKFx8` |
| platform_station_id (точка А) | `fbed3aa1-2cc6-4370-ab4d-59c5cc9bb924` |

### Сколько времени на подтверждение заявки?

**10 минут.** Если клиенту нужно больше времени, не создавайте заявку до получения подтверждения. Для предварительного расчёта используйте [Предварительная оценка стоимости](yandex_delivery_other_day_pricing_calculator.md) и [Получение интервалов](yandex_delivery_other_day_offersinfo_post.md).

### Как завести собственную станцию?

Обратитесь к персональному менеджеру.

### Как настроить отгрузки на своей станции?

Персональный менеджер определит график забора с вашего склада.

### Можно ли самостоятельно привозить заказ на сортировочный центр?

Да. Получите точки самопривоза через метод [Получение точек самопривоза и ПВЗ](yandex_delivery_other_day_pickup_points_list.md). Привезти заказ можно до **21:00**. Индивидуальные условия — через менеджера.

### Какие ограничения по габаритам и весу?

Зависят от способа доставки. Подробнее: [Правила отгрузки](https://yandex.com/support/delivery-profile/ru/other-day/rules).

### Какие ограничения по страховой стоимости?

Максимум **250 000 рублей**.

### В каком часовом поясе передавать время?

Формат UNIX или UTC — **GMT+0**. Ответ также в GMT+0.

### Что такое штрихкод? Какие требования?

Уникальный идентификатор товара в заказе. Допустимые символы: `a-z`, `A-Z`, `0-9`, `-`, `/`, `\`, `_`. В одном заказе может быть несколько товаров с одинаковым штрихкодом. Подробнее: [Маркировать товар](https://yandex.com/support/delivery-profile/ru/other-day/mark-rules).

### Что такое АПП (акт приема-передачи)?

Документ, подтверждающий факт отгрузки курьером ваших товаров. Генерация: метод [Получение акта приема-передачи](yandex_delivery_other_day_handover_act.md). Можно также использовать шаблоны документов вашей компании.

## Вопросы, связанные с ошибками методов API

### Not authorized request

| Причина | Решение |
|---------|---------|
| Сменили пароль в ЛК | Актуализируйте токен |
| Перепутали окружение | Тестовый токен — для тестового хоста, боевой — для боевого |
| Некорректный формат | Проверьте заголовок `Authorization: Bearer YOUR_TOKEN` |

### Delivery interval in the past

Проверьте, что интервал доставки указан в будущем.

### Delivery interval is too narrow

Ошибка возникает, если в регионе доставки нет переданного интервала. Например, при доставке с 9:00 до 18:00 интервал 9:00–13:00 вернёт ошибку.

### Cannot deliver

Система не смогла выстроить маршрут. Проверьте:

| Тип доставки | Что проверить |
|--------------|---------------|
| До двери | Корректный интервал доставки через метод создания заявки |
| До ПВЗ/постамата | Формат интервалов: `{"from": "2022-06-16", "to": "2022-06-16"}` |

Если ошибка сохраняется — обратитесь в [службу поддержки](https://yandex.com/support/delivery-profile/ru/api/other-day/troubleshooting).

### Inconsistency billing details

Ошибка оценки стоимости. Проверьте единицы измерения стоимости товаров (должны быть в **копейках**).

### Incorrect station platform ID

| Причина | Описание |
|---------|----------|
| Неверная среда | Боевая станция в тестовом запросе или наоборот |
| Не закреплена | Станция не привязана к пользователю или не существует |

### Incorrect JSON content. Please check the formatting

Техническая ошибка парсинга. Проверьте корректность JSON.

### There is place with no items / There are items with no place

У каждого грузоместа должен быть хотя бы один товар, и у каждого товара — своё грузоместо.

### Cannot parse ... info

Техническая ошибка парсинга тела запроса. Проверьте корректность JSON.

### There already was request with such code within this employer

`operator_request_id` уникален для каждого заказа. Повторное создание с тем же ID вызовет эту ошибку.
