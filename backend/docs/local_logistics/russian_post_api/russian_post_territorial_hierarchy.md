# Иерархия территориального деления России в API Почты России

## Введение

API Почты России («Отправка») использует упрощённую адресную модель из **7 уровней**, которая является проекцией полной государственной иерархии территориального деления РФ (ОКТМО/ФИАС) на нужды почтовой доставки. Этот документ описывает, как официальные уровни деления отображаются в поля API, какие уровни схлопываются, а какие отсутствуют.

---

## Полная государственная иерархия vs. API Почты России

```
Государственная иерархия (ОКТМО + ФИАС/ГАР)          API Почты России
─────────────────────────────────────────────          ─────────────────
Страна (ISO 3166-1: RU)                               — (не передаётся, подразумевается)
  │
  ├─ Субъект Федерации (89 шт.)                        → region          ← Уровень 1
  │    область, край, республика, город фед.
  │    значения, автономный округ
  │
  ├─ Муниц. район / Городской округ (~2 340 шт.)      → area / district ← Уровень 2
  │    ОКТМО: цифра 3 (P1) = тип
  │
  ├─ Поселение (городское / сельское)                  ─ ОТСУТСТВУЕТ ─   ← Схлопнут
  │    ОКТМО: цифра 6 (P2) = тип
  │
  ├─ Населённый пункт (~155 000 шт.)                  → place / settlement ← Уровень 3
  │    город, ПГТ, село, деревня, хутор
  │    ОКТМО: цифры 9–11 (11-значный код)
  │
  ├─ Элемент планировочной структуры                   → location        ← Уровень 4
  │    микрорайон, СНТ, квартал                         (необязательный)
  │    ГАР: уровень 7
  │
  ├─ Улично-дорожная сеть                              → street          ← Уровень 5
  │    улица, проспект, переулок, бульвар
  │    ГАР: уровень 8
  │
  ├─ Земельный участок (ГАР: уровень 9)               ─ ОТСУТСТВУЕТ ─   ← Не используется
  │
  ├─ Здание / сооружение                               → house + building ← Уровень 6
  │    дом, корпус, строение, литера                      + corpus + letter
  │    ГАР: уровень 10                                   + slash + vladenie
  │
  ├─ Помещение                                         → room / office   ← Уровень 7
  │    квартира, офис, гостиница                          / hotel
  │    ГАР: уровень 11
  │
  ├─ Комната в помещении (ГАР: уровень 12)            ─ ОТСУТСТВУЕТ ─   ← Не используется
  └─ Машино-место (ГАР: уровень 17)                   ─ ОТСУТСТВУЕТ ─   ← Не используется
```

**Ключевые отличия от полной иерархии:**

- **Поселение** (городское/сельское) — уровень 3 ОКТМО — в API отсутствует; населённый пункт указывается напрямую
- **Земельный участок**, **комната в помещении**, **машино-место** — уровни ГАР 9, 12, 17 — не нужны для почтовой доставки
- **Федеральный округ** (ЦФО, СЗФО и т.д.) — не является уровнем АТД, в API отсутствует

## Детализация каждого уровня

### Уровень 1: Регион (`region` / `region-to` / `region-from`)

Субъект Российской Федерации — верхний уровень территориального деления в API.

| Тип субъекта        | Кол-во | ISO 3166-2 | Пример               |
| ------------------- | ------ | ---------- | -------------------- |
| Республика          | 22     | `RU-TA`    | Республика Татарстан |
| Край                | 9      | `RU-KDA`   | Краснодарский край   |
| Область             | 48     | `RU-MOS`   | Московская область   |
| Город фед. значения | 3      | `RU-MOW`   | Москва               |
| Автономная область  | 1      | `RU-YEV`   | Еврейская АО         |
| Автономный округ    | 4      | `RU-KHM`   | Ханты-Мансийский АО  |
| **Итого по РФ**     | **89** |            |                      |

**В API:** значение `region` — строковое название (например, `"Свердловская"`, `"Москва"`, `"Татарстан"`).

**Классификатор:** ОКТМО — первые 2 цифры (`XX`).

> **Особый случай:** Для городов федерального значения (Москва, Санкт-Петербург, Севастополь) значения `region` и `place` совпадают. Например: `"region": "Москва"`, `"place": "Москва"`.

### Уровень 2: Район (`area` / `area-to` / `area-from` / `district`)

Муниципальный район, городской округ или муниципальный округ.

| Тип МО                     | ОКТМО P1 | Пример            |
| -------------------------- | -------- | ----------------- |
| Муниципальный район        | 6        | Одинцовский район |
| Городской округ            | 7        | ГО Екатеринбург   |
| Муниципальный округ        | 5        | МО Алапаевский    |
| Внутригородская территория | 3        | МО Арбат (Москва) |

**Классификатор:** ОКТМО — цифра 3 (P1) определяет тип, цифры 4–5 — порядковый номер.

**В API:**

- Модель заказа (OrderV1): поле `area` (отправитель) / `area-to` (получатель)
- Модель почтового отделения (PostOffice): поле `district`
- Модель нормализованного адреса (NormalizedAddress): поле `area`
- Может быть `null` для городских округов и городов федерального значения

> **Несоответствие имён:** один и тот же уровень называется `area` в заказе, `district` в PostOffice. Это особенность API — маппинг при конвертации данных между моделями.

### Уровень 3: Населённый пункт (`place` / `place-to` / `place-from` / `settlement`)

Город, посёлок городского типа, посёлок, село, деревня, станица, хутор, аул.

| Тип                  | ОКТМО (цифры 9–11) | Пример          |
| -------------------- | ------------------ | --------------- |
| Город                | 001–049            | г. Екатеринбург |
| ПГТ                  | 051–099            | пгт Лесной      |
| Сельские нас. пункты | 101–999            | д. Петрово      |

**В API:**

- Модель заказа: `place` (отправитель) / `place-to` (получатель)
- Модель PostOffice: `settlement`
- Модель NormalizedAddress: `place`

> **Уровень «Поселение» (городское/сельское) пропущен:** в государственной иерархии между районом и населённым пунктом существует уровень поселения (ОКТМО цифры 6–8). API Почты России его не использует — населённый пункт указывается напрямую после района.

### Уровень 4: Микрорайон (`location` / `location-to`) — необязательный

Элемент планировочной структуры: микрорайон, квартал, жилой комплекс, СНТ.

- Соответствует **уровню 7 ГАР** (AS_ADDR_OBJ)
- Заполняется только для крупных городов с чёткой внутренней структурой
- Может быть `null` в большинстве случаев

### Уровень 5: Улица (`street` / `street-to` / `street-from`)

Элемент улично-дорожной сети: улица, проспект, переулок, бульвар, площадь, набережная, шоссе.

- Соответствует **уровню 8 ГАР** (AS_ADDR_OBJ)

### Уровень 6: Дом и строение

Здание или сооружение — соответствует **уровню 10 ГАР** (AS_HOUSES).

| Поле API   | Описание в API             | Описание                        |
| ---------- | -------------------------- | ------------------------------- |
| `house`    | Часть адреса: Номер здания | Номер дома                      |
| `building` | Часть здания: Строение     | Строение (стр.)                 |
| `corpus`   | Часть здания: Корпус       | Корпус (корп.)                  |
| `letter`   | Часть здания: Литера       | Буквенный суффикс дома (лит. А) |
| `slash`    | Часть здания: Дробь        | Дробная часть адреса (д. 15/2)  |
| `vladenie` | Часть адреса: Владение     | Владение (вл.)                  |

### Уровень 7: Помещение

Помещение — соответствует **уровню 11 ГАР** (AS_APARTMENTS).

| Поле API | Описание в API                | Описание           |
| -------- | ----------------------------- | ------------------ |
| `room`   | Часть здания: Номер помещения | Квартира / комната |
| `office` | Часть здания: Офис            | Офис               |
| `hotel`  | Название гостиницы            | Гостиница          |

## Почтовый индекс (`index` / `postal-code`)

6-значный цифровой код, привязанный к конкретному отделению почтовой связи (ОПС).

**Структура индекса:**

```
  X  XX  XXX
  │   │    │
  │   │    └── Номер отделения в рамках УФПС
  │   └─────── Код УФПС (региональное управление)
  └─────────── Код почтовой зоны (тарифная зона)
```

Почтовый индекс — это **не часть территориальной иерархии**, а **ключ связи** между адресом и отделением почтовой связи (ОПС). Один населённый пункт может содержать множество индексов (один на каждое ОПС).

**API-эндпоинты для работы с индексами:**

| Эндпоинт                                   | Метод | Описание                        | Параметры                                           |
| ------------------------------------------ | ----- | ------------------------------- | --------------------------------------------------- |
| `/postoffice/1.0/settlement.offices.codes` | GET   | Все индексы в населённом пункте | `settlement`, `region`, `district`                  |
| `/postoffice/1.0/{postal-code}/settlement` | GET   | Населённый пункт по индексу     | `postal-code` → `{postal-code, region, settlement}` |

**Пример ответа** `/postoffice/1.0/{postal-code}/settlement`:

```json
{
  "postal-code": "101000",
  "region": "Москва",
  "settlement": "Москва"
}
```

> Ответ содержит только **2 уровня территории**: регион и населённый пункт. Район не возвращается — он известен только модели PostOffice (поле `district`).

## Специальные типы адресов (`AddressType`)

Помимо стандартной иерархии, API поддерживает альтернативные типы адресации, при которых уровни 5–7 заменяются одним полем:

| Тип       | Описание                           | Используемые поля               |
| --------- | ---------------------------------- | ------------------------------- |
| `DEFAULT` | Стандартный (улица, дом, квартира) | Полная иерархия уровней 1–7     |
| `PO_BOX`  | Абонентский ящик                   | Уровни 1–3 + `num-address-type` |
| `DEMAND`  | До востребования                   | Уровни 1–3 + `num-address-type` |
| `UNIT`    | Войсковая часть                    | Уровни 1–3 + `num-address-type` |

Для типов `PO_BOX`, `DEMAND`, `UNIT` поля `street`, `house`, `room` и т.д. не используются — вместо них заполняется `num-address-type` с номером ящика / части.

## Нормализация адреса

Эндпоинт `POST /1.0/clean/address` разбирает произвольную строку адреса по уровням иерархии и валидирует по справочнику **ФИАС** (Федеральная информационная адресная система, ранее КЛАДР).

### Входные данные

```json
[{ "id": "adr-1", "original-address": "Москва, Тверская 1" }]
```

### Результат — адрес, разложенный по уровням

```json
[
  {
    "id": "adr-1",
    "original-address": "Москва, Тверская 1",
    "index": "125009",
    "region": "Москва",
    "area": null,
    "place": "Москва",
    "location": null,
    "street": "Тверская",
    "house": "1",
    "building": null,
    "corpus": null,
    "letter": null,
    "slash": null,
    "room": null,
    "quality-code": "GOOD",
    "validation-code": "VALIDATED"
  }
]
```

### Коды качества (`AddressNormalizationQuality`)

Код качества указывает, до какого уровня иерархии удалось распознать адрес:

| Код          | Описание                            | Нераспознанный уровень |
| ------------ | ----------------------------------- | ---------------------- |
| `GOOD`       | Пригоден для почтовой доставки      | — (все уровни ОК)      |
| `ON_DEMAND`  | До востребования                    | —                      |
| `POSTAL_BOX` | Абонентский ящик                    | —                      |
| `UNDEF_01`   | Не определён регион                 | **Уровень 1** (region) |
| `UNDEF_02`   | Не определён город/населённый пункт | **Уровень 3** (place)  |
| `UNDEF_03`   | Не определена улица                 | **Уровень 5** (street) |
| `UNDEF_04`   | Не определён дом                    | **Уровень 6** (house)  |
| `UNDEF_05`   | Не определена квартира/офис         | **Уровень 7** (room)   |
| `UNDEF_06`   | Не определено (общий)               | —                      |
| `UNDEF_07`   | Иностранный адрес                   | —                      |

> **Обратите внимание:** уровни 2 (район) и 4 (микрорайон) не имеют отдельных кодов ошибки — они необязательны для доставки.

### Коды валидации (`AddressNormalizationValidation`)

| Код                                           | Описание                              | Действие         |
| --------------------------------------------- | ------------------------------------- | ---------------- |
| `CONFIRMED_MANUALLY`                          | Подтверждено вручную                  | Принять          |
| `VALIDATED`                                   | Уверенное распознавание               | Принять          |
| `OVERRIDDEN`                                  | Адрес перезаписан из справочника ФИАС | Принять          |
| `NOT_VALIDATED_HAS_NO_KLADR_RECORD`           | Не найден в КЛАДР/ФИАС                | Требует проверки |
| `NOT_VALIDATED_HAS_ASSUMPTION`                | Сделано допущение                     | Требует проверки |
| `NOT_VALIDATED_HAS_UNPARSED_PARTS`            | Есть нераспознанные части             | Требует проверки |
| `NOT_VALIDATED_HAS_NO_MAIN_POINTS`            | Отсутствуют основные компоненты       | Требует проверки |
| `NOT_VALIDATED_HAS_NUMBER_STREET_ASSUMPTION`  | Допущение по названию улицы           | Требует проверки |
| `NOT_VALIDATED_HOUSE_WITHOUT_STREET_OR_NP`    | Дом без улицы или населённого пункта  | Требует проверки |
| `NOT_VALIDATED_HOUSE_EXTENSION_WITHOUT_HOUSE` | Строение/корпус без номера дома       | Требует проверки |
| `NOT_VALIDATED_HAS_AMBI`                      | Неоднозначное соответствие            | Требует проверки |
| `NOT_VALIDATED_EXCEDED_HOUSE_NUMBER`          | Номер дома слишком большой            | Требует проверки |
| `NOT_VALIDATED_INCORRECT_HOUSE`               | Неверный номер дома                   | Требует проверки |
| `NOT_VALIDATED_INCORRECT_HOUSE_EXTENSION`     | Неверное строение/корпус              | Требует проверки |
| `NOT_VALIDATED_FOREIGN`                       | Иностранный адрес                     | Не валидируется  |
| `NOT_VALIDATED_DICTIONARY`                    | Не сопоставлен со справочником        | Требует проверки |

**Бизнес-правило BR-008:** адрес считается корректным только при **одновременном** выполнении:

- `quality-code` ∈ {`GOOD`, `POSTAL_BOX`, `ON_DEMAND`, `UNDEF_05`}
- `validation-code` ∈ {`VALIDATED`, `OVERRIDDEN`, `CONFIRMED_MANUALLY`}

## Соответствие полей в разных моделях API

Одни и те же уровни иерархии имеют **разные имена полей** в разных моделях:

| Уровень             | OrderV1 (отправитель)       | OrderV1 (получатель) | NormalizedAddress | PostOffice    |
| ------------------- | --------------------------- | -------------------- | ----------------- | ------------- |
| 1. Регион           | `address-from.region`       | `region-to`          | `region`          | `region`      |
| 2. Район            | `address-from.area`         | `area-to`            | `area`            | `district`    |
| 3. Населённый пункт | `address-from.place`        | `place-to`           | `place`           | `settlement`  |
| 4. Микрорайон       | `address-from.location`     | `location-to`        | `location`        | —             |
| 5. Улица            | `address-from.street`       | `street-to`          | `street`          | (в `address`) |
| 6. Дом              | `address-from.house`        | `house-to`           | `house`           | (в `address`) |
| 6. Корпус           | `address-from.corpus`       | `corpus-to`          | `corpus`          | —             |
| 6. Строение         | `address-from.building`     | `building-to`        | `building`        | —             |
| 6. Литера           | `address-from.letter`       | `letter-to`          | `letter`          | —             |
| 6. Дробь            | `address-from.slash`        | `slash-to`           | `slash`           | —             |
| 6. Владение         | `address-from.vladenie`     | `vladenie-to`        | —                 | —             |
| 7. Помещение        | `address-from.room`         | `room-to`            | `room`            | —             |
| 7. Офис             | `address-from.office`       | `office-to`          | —                 | —             |
| 7. Гостиница        | `address-from.hotel`        | `hotel-to`           | `hotel`           | —             |
| Почтовый индекс     | `address-from.index`        | `index-to`           | `index`           | `postal-code` |
| Тип адреса          | `address-from.address-type` | `address-type-to`    | `address-type`    | —             |

> **Закономерности именования:**
>
> - Отправитель: `address-from.{field}` (вложенный объект)
> - Получатель: `{field}-to` (плоские поля верхнего уровня)
> - PostOffice: собственные имена (`district` вместо `area`, `settlement` вместо `place`)

## Организационная иерархия почтовой сети

Помимо территориального деления адресов, Почта России имеет собственную организационную иерархию:

```
Почта России (головной офис — АО «Почта России»)
  └── УФПС (Управление федеральной почтовой связи) — по регионам
        └── ОПС (Отделение почтовой связи) — конкретное почтовое отделение
```

| Уровень | Описание                                           | Поле в API (PostOffice) | Идентификатор             |
| ------- | -------------------------------------------------- | ----------------------- | ------------------------- |
| УФПС    | Региональное управление федеральной почтовой связи | `ufps-code`             | Код УФПС                  |
| ОПС     | Отделение почтовой связи                           | `postal-code`           | 6-значный почтовый индекс |

**Типы ОПС** (поле `type-code`):

| Код    | Описание                             |
| ------ | ------------------------------------ |
| `ГОПС` | Городское отделение почтовой связи   |
| `СОПС` | Сельское отделение почтовой связи    |
| `ПОПС` | Передвижное отделение почтовой связи |

**Модель PostOffice** содержит территориальную привязку отделения:

```json
{
  "postal-code": "101000",
  "region": "Москва",
  "district": "Центральный",
  "settlement": "Москва",
  "address-source": "ФИАС",
  "ufps-code": "101000",
  "type-code": "ГОПС",
  "latitude": 55.761168,
  "longitude": 37.632684
}
```

## Связь с классификаторами

| Классификатор           | Уровни, покрываемые в API                 | Как используется                                                                                                  |
| ----------------------- | ----------------------------------------- | ----------------------------------------------------------------------------------------------------------------- |
| **ОКТМО** (ОК 033-2013) | Регион → Район → (Поселение) → Нас. пункт | Не используется напрямую; структура адреса повторяет ОКТМО, но без кодов                                          |
| **ФИАС/ГАР**            | Все 7 уровней + валидация                 | Основной справочник. Эндпоинт нормализации валидирует адреса по ФИАС. Поле `address-source: "ФИАС"` в PostOffice  |
| **КЛАДР**               | Legacy                                    | Устаревший справочник. Упоминается в коде валидации `NOT_VALIDATED_HAS_NO_KLADR_RECORD` как историческое название |
| **ISO 3166-2:RU**       | Только уровень 1 (регион)                 | Не используется напрямую в API; 83 кода ISO vs 89 субъектов по РФ                                                 |

## Особые случаи территориального деления в контексте API

| Ситуация                                             | Как отражается в API                                                                                                             |
| ---------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------- |
| Москва, Питер, Севастополь (города фед. значения)    | `region` = `place` (например: `"region": "Москва"`, `"place": "Москва"`). `area`/`district` может содержать округ                |
| Крупные городские округа (Екатеринбург, Новосибирск) | `area` = `null` (нет муниципального района). `place` = название города                                                           |
| ЗАТО (закрытые города: Саров, Озёрск)                | Отдельный городской округ, `area` = `null`                                                                                       |
| Международные отправления                            | `index-to` заменяется на `str-index-to` (буквенно-цифровой), `mail-direct` = код страны                                          |
| Автономные округа внутри областей                    | И округ, и область — отдельные субъекты уровня 1. Хантымансийский АО имеет свой `region`, несмотря на вхождение в Тюменскую обл. |
| Населённый пункт без улицы                           | Допустимо при `quality-code: UNDEF_03`. В малых сёлах адрес может быть: регион + нас. пункт + дом                                |
