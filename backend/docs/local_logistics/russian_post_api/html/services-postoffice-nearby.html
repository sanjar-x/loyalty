Поиск почтовых отделений по координатам

Введите токен авторизации приложения и ключ авторизации пользователя, тогда значения заголовков будут сгенерированы автоматически.

Токен авторизации
Ключ авторизации
latitude
Широта
longitude
Долгота
top
Количество ближайших почтовых отделений в результате поиска (Опционально)
filter
Дополнительное ограничение по времени работы для поиска ОПС.
search-radius
Радиус для поиска (в километрах) (Опционально)
current-date-time
Текущее клиентское дата-время в таймзоне клиента. Формат: yyyy-MM-dd'T'HH:mm:ss. Данный параметр используется для определения отделений, работающих в данный момент, т.е. если эти данные нужны, параметр передавать обязательно!
hide-private
Исключать не публичные отделения (Опционально). По-умолчанию не исключать (false).
filter-by-office-type
Фильтр по типам объектов в ответе. true: ГОПС, СОПС, ПОЧТОМАТ. false: все. Значение по-умолчанию - true.
yandex-address
Адрес в том формате, в котором возвращает его сервис Яндекса для адреса, указанного пользователем. Пример: Санкт-Петербург, улица Победы, 15к1. Параметр необходим для определения является ли переданный адрес точным адресом отделения. Требует также заполненного параметра geoObject.
geo-object
JSON-строка, содержащая объект GeoObject, получаемый для адреса в сервисе Яндекса. См. api.yandex.ru. Требует также заполненного параметра 'yandex-address'.
Параметры запроса
Локальный URL /postoffice/1.0/nearby{{ url }}
Тип GET
Заголовки запроса
Authorization AccessToken {{ token }}
X-User-Authorization Basic {{ key }}
Content-Type application/json;charset=UTF-8
