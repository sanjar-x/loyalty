# Возврат заказов в "Новые"

Метод переводит заказы из партии в раздел Новые. Партия должна быть в статусе CREATED.

## Параметры запроса

| Параметр | Значение |
|----------|----------|
| Локальный URL | **/1.0/user/backlog** |
| Тип | **POST** |

## Заголовки запроса

| Заголовок | Значение |
|-----------|----------|
| **Authorization** | **AccessToken {token}** |
| **X-User-Authorization** | **Basic {key}** |
| **Content-Type** | **application/json;charset=UTF-8** |

## Тело запроса

```json
Array[string]
```

Список внутренних идентификаторов заказов.

## Ответ на запрос

```json
{
  "errors": [
    {
      "error-code": "UNDEFINED",
      "error-details": "string",
      "position": 0
    }
  ],
  "result-ids": [
    0
  ]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| errors | Массив (Опционально) | Список ошибок |
| errors[].error-code | Строка (Опционально) | Код ошибки |
| errors[].error-details | Строка (Опционально) | Описание ошибки |
| errors[].position | Целое число (Опционально) | Индекс в исходном массиве |
| result-ids | Массив (Опционально) | Список успешно обработанных внутренних идентификаторов отправлений |
