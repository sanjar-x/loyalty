Проверка и нормализация адреса с помощью SoapUI

Цель данного раздела:
  • проверить авторизацию на API сервисе (здесь будут указаны реальные токен авторизации и ключ авторизации);
  • проверить возможности метода нормализации адреса.
Здесь рассматриваются базовые подходы к созданию REST запросов, которые могут быть использованы для отладки других сервисов API Онлайн-сервиса «Отправка».
Сам метод нормализации адреса может быть вам полезен, когда требуется разделить входящий адрес получателя на составляющие адресные части.

1 шаг: Скачайте и установите SoapUI

1. Перейдите по ссылке SoapUI.
2. Скачайте "SoapUI OpenSource" бесплатную версию.
3. Установите SoapUI.

2 шаг: Запустите SoapUI и создайте REST проект

После запуска:
1. Создайте "REST" проект
2. Укажите ссылку "{{exampleHost}}" на API Онлайн-сервиса «Отправка».
3. Нажмите на кнопку "OK".



3 шаг: Создайте новый запрос

После запуска:
1. Выделите ресурс, кликните на нём правой кнопкой мыши и выберите меню "New REST Resource"
2. Укажите локальную ссылку "/1.0/clean/address". Данная ссылка взята из метода нормализации адреса. Ссылки для других методов уточняйте в документации.
3. Нажмите на кнопку "OK".



4 шаг: Отредактируйте запрос

1. Выберите ресурс
2. В открывшемся окне укажите тип запроса POST. Данный тип взят из метода нормализации адреса. Типы для других методов уточняйте в документации. Это могут быть следующие типы: "GET", "POST", "PUT" или "DELETE".
3. Нажмите на закладку "Headers" и добавьте 3 заголовка:
  • Добавьте параметр авторизации приложения Authorization и укажите префикс AccessToken, символ "пробела" и токен авторизации приложения;
  • Добавьте параметр авторизации пользователя X-User-Authorization и укажите префикс Basic, символ "пробела" и ключ авторизации пользователя;
  • Добавьте параметр формата ожидаемого результата Content-Type и укажите application/json;charset=UTF-8
4. В теле запроса укажите один или несколько адресов для нормализации, согласно формату метода сервиса нормализации адреса.

Пример:
Мы должны нормализовать два адреса: "Москва, Варшавское шоссе, 37" и "ул. Мясницкая, д. 26, г. Москва".
Для этого:
1. Для каждого адреса мы готовим JSON структуру: { "id": "adr 1", "original-address": "Москва, Варшавское шоссе, 37" } и { "id": "adr 2", "original-address": "ул. Мясницкая, д. 26, г. Москва" } , где
  • id - уникальный признак этого адреса, в рамках одного запроса. Т.к. нормализация двух адресов будет проходит в рамках одного запроса, то выбираем произвольные различающиеся признаки, например: adr 1 и adr 2;
  • original-address - адрес, который мы нормализуем.
2. Эти JSON структуры мы должны объединить в группу.
Для обозначения группы используются "квадратные скобки" ( [ и ] ).
Для разделения элементов внутри группы используется "запятая" ( , ).

Таким образом, мы получаем окончательное тело запроса:
[
    {
        "id": "adr 1",
        "original-address": "Москва, Варшавское шоссе, 37"
    },
    {
        "id": "adr 2",
        "original-address": "ул. Мясницкая, д. 26, г. Москва, 1"
    }
]
Внимание! Если метод групповой, то необходимо создавать группу даже для одного элемента JSON структуры.
Пример:
[
    {
        "id": "adr 2",
        "original-address": "Москва, Варшавское шоссе, 37"
    }
]



5 шаг: Выполните запрос

1. Выберите тип ожидаемого ответа JSON
2. Нажмите на знак запуска.
3. Проконтролируйте полученный результат.
