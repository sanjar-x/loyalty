Создание партии из N заказов

Автоматически создает партию и переносит указанные подготовленные заказы в эту партию. Если заказы относятся к разным типам и категориям – создается несколько партий. Заказы распределяются по соответствующим партиям. Каждому перенесенному заказу автоматически присваивается ШПИ.

Введите токен авторизации приложения, ключ авторизации пользователя и другие параметры, тогда значения заголовков и параметров будут сгенерированы автоматически.

Токен авторизации
Ключ авторизации
sending-date (Опционально)
Дата сдачи в почтовое отделение (yyyy-MM-dd)
timezone-offset (Опционально)
Смещение даты сдачи от UTC в секундах
use-online-balance
(Опционально)
Признак использования онлайн баланса
Параметры запроса
Локальный URL /1.0/user/shipment{{ buildUrlParams({ sendingDate: { value: sendingDate, prefix: 'sending-date' }, timezoneOffset: { value: timezoneOffset, prefix: 'timezone-offset' }, useOnlineBalance: { value: useOnlineBalance, prefix: 'use-online-balance' } }) }}
Тип POST
Заголовки запроса
Authorization AccessToken {{ token }}
X-User-Authorization Basic {{ key }}
Content-Type application/json;charset=UTF-8
