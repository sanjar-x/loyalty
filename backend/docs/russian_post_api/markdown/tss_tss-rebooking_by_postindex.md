# Перебронирование интервала доставки

Метод бронирует новый таймслот для РПО на новую дату по идентификатору таймслота и ШПИ РПО. Ранее забронированный интервал помечается как удаленный, емкость интервала увеличивается на 1.

## Параметры запроса

| Параметр | Значение |
|----------|----------|
| Локальный URL | **/external/v1/rebooking-by-postindex** |
| Тип | **POST** |

## Заголовки запроса

| Заголовок | Значение |
|-----------|----------|
| **Authorization** | **AccessToken {token}** |
| **X-User-Authorization** | **Basic {key}** |
| **Content-Type** | **application/json;charset=UTF-8** |

## Тело запроса

```json
{
  "uuidOrBarcode": "string",
  "timeslotId": 0,
  "contractNumber": "string",
  "plannedShippingDate": "string"
}
```

| Параметр | Тип | Описание |
|----------|-----|----------|
| uuidOrBarcode | Строка | UUID предварительного бронирования или ШПИ (максимальная длина 50 символов) |
| timeslotId | Целое число | Идентификатор временного интервала |
| contractNumber | Строка | Номер договора (максимальная длина 50 символов) |
| plannedShippingDate | Строка | Плановая дата в формате: yyyy-MM-dd |

## Ответ на запрос (ошибка)

```json
{
  "code": "string",
  "message": "string",
  "errors": [
    {
      "field": "string",
      "message": "string"
    }
  ]
}
```

| Параметр | Тип | Описание |
|----------|-----|----------|
| code | Строка | Текстовый идентификатор ошибки |
| message | Строка | Подробное описание ошибки поля входных данных |
| errors | Массив | Возвращается при наличии ошибок в полях входных данных |
| errors[].field | Строка | Текстовый идентификатор поля входных данных |
| errors[].message | Строка | Подробное описание ошибки поля входных данных |
