# Иерархия территориального деления России в API Yandex Delivery

## Введение

API Yandex Delivery («Другой день») использует принципиально иной подход к адресации по сравнению с государственными системами (ОКТМО/ФИАС) и API Почты России. Вместо строгой иерархической декомпозиции адреса Yandex применяет **двухуровневую модель**:

1. **Плоская строка адреса** (`address` / `full_address`) — основной инструмент маршрутизации, геокодируется внутренним геокодером Яндекса
2. **Структурированный объект `LocationDetails`** — детализация для отображения и метаданных, заполняется геокодером

Yandex **не валидирует адреса по ФИАС/КЛАДР**. Вместо этого используется собственный геокодер и система идентификации населённых пунктов `geo_id` (GeoBase).

## Государственная иерархия vs. API Yandex Delivery

```
Государственная иерархия (ОКТМО + ФИАС/ГАР)          API Yandex Delivery
─────────────────────────────────────────────          ────────────────────
Страна (ISO 3166-1: RU)                               → country
  │
  ├─ Субъект Федерации (89 шт.)                        → region               ← LocationDetails
  │    область, край, республика, город фед.
  │    значения, автономный округ
  │
  ├─ Муниц. район / Городской округ (~2 340 шт.)      → subRegion (≈)        ← LocationDetails
  │    ОКТМО: цифра 3 (P1) = тип                        (заполняется геокодером,
  │                                                       не всегда совпадает с ОКТМО)
  │
  ├─ Поселение (городское / сельское)                  ─ ОТСУТСТВУЕТ ─
  │    ОКТМО: цифра 6 (P2)
  │
  ├─ Населённый пункт (~155 000 шт.)                  → locality + geo_id    ← LocationDetails
  │    город, ПГТ, село, деревня                         (geo_id — внутренний ID Яндекса)
  │
  ├─ Элемент планировочной структуры                   ─ ОТСУТСТВУЕТ ─
  │    микрорайон, СНТ (ГАР: уровень 7)
  │
  ├─ Улично-дорожная сеть                              → street               ← LocationDetails
  │    улица, проспект, переулок
  │    (ГАР: уровень 8)
  │
  ├─ Земельный участок (ГАР: уровень 9)               ─ ОТСУТСТВУЕТ ─
  │
  ├─ Здание / сооружение                               → house + housing      ← LocationDetails
  │    дом, корпус, строение                               + building
  │    (ГАР: уровень 10)
  │
  ├─ Помещение                                         → apartment            ← LocationDetails
  │    квартира                                            (НЕ в строке адреса!)
  │    (ГАР: уровень 11)
  │
  ├─ Комната в помещении (ГАР: уровень 12)            ─ ОТСУТСТВУЕТ ─
  └─ Машино-место (ГАР: уровень 17)                   ─ ОТСУТСТВУЕТ ─

Дополнительно:
  • Координаты: latitude + longitude (GeoPoint / CustomLocation)
  • Идентификатор ПВЗ: platform_station_id (UUID)
  • Почтовый индекс: postal_code (информационное поле)
  • geo_id: внутренний идентификатор Яндекса (GeoBase)
```

**Ключевые отличия от государственной иерархии:**

- **Поселение**, **микрорайон**, **земельный участок**, **комната**, **машино-место** — отсутствуют
- **subRegion** ≠ район по ОКТМО (заполняется геокодером, может отличаться)
- **Нет валидации по ФИАС/КЛАДР** — адрес геокодируется, а не сопоставляется со справочником
- **Координаты — первичны**: адрес преобразуется в координаты для маршрутизации

## Два способа указания адреса в API

Yandex Delivery предлагает **два принципиально разных** способа задания точки назначения, в зависимости от эндпоинта:

### Способ 1: Плоская строка (`SimpleDestinationNode`)

Используется в эндпоинтах предварительной оценки (1.01, 1.02, 1.03):

```json
{
  "destination": {
    "address": "Санкт-Петербург, Большая Монетная улица, 1к1А"
  }
}
```

| Поле                  | Тип    | Описание                                                           |
| --------------------- | ------ | ------------------------------------------------------------------ |
| `address`             | string | Адрес в формате «Город, улица, дом». **Без квартиры, без индекса** |
| `platform_station_id` | string | UUID пункта выдачи (альтернатива `address`)                        |

> **Бизнес-правило BR-009:** Формат адреса: «Город, улица, дом». Части разделяются запятыми. Нельзя включать: почтовый индекс, номер квартиры (квартира → в `details.apartment`), домофон/подъезд (→ в `comment`).

### Способ 2: Структурированный объект (`CustomLocation` + `LocationDetails`)

Используется в эндпоинтах создания заказа (3.01, 3.11):

```json
{
  "destination": {
    "type": "custom_location",
    "custom_location": {
      "latitude": 55.666624,
      "longitude": 37.51573,
      "details": {
        "geoId": 213,
        "country": "Россия",
        "region": "Москва",
        "subRegion": "Московская область",
        "locality": "Москва",
        "street": "Пролетарский проспект",
        "house": "19",
        "housing": "1",
        "building": "1",
        "apartment": "2",
        "comment": "Станция метро Щукинская...",
        "full_address": "Москва, Пролетарский проспект, 19",
        "postal_code": "123182"
      }
    }
  }
}
```

### Способ 3: Идентификатор ПВЗ (`platform_station`)

Для самовывоза — вместо адреса указывается UUID пункта выдачи:

```json
{
  "destination": {
    "type": "platform_station",
    "platform_station": {
      "platform_id": "01946f4f013c7337874ec2fb848a58a4"
    }
  }
}
```

## Модель `LocationDetails` — структурированный адрес

Основная модель детализации адреса, используемая в `CustomLocation.details` и `PickupStation.address`:

```
LocationDetails
├── geoId          — идентификатор населённого пункта (GeoBase Яндекса)
├── country        — страна
├── region         — регион (субъект РФ)
├── subRegion      — подрегион (≈ район)
├── locality       — населённый пункт
├── street         — улица
├── house          — номер дома
├── housing        — корпус
├── building       — строение
├── apartment      — квартира
├── comment        — комментарий (домофон, подъезд, этаж)
├── full_address   — полный адрес одной строкой
└── postal_code    — почтовый индекс
```

| Поле           | Тип    | Обязательное | Описание                        | Гос. уровень  | Пример                                |
| -------------- | ------ | ------------ | ------------------------------- | ------------- | ------------------------------------- |
| `geoId`        | number | нет          | ID населённого пункта (GeoBase) | —             | `213` (Москва)                        |
| `country`      | string | нет          | Страна                          | ISO 3166-1    | `"Россия"`                            |
| `region`       | string | нет          | Регион (субъект РФ)             | ОКТМО ур. 1   | `"Москва"`                            |
| `subRegion`    | string | нет          | Подрегион                       | ≈ ОКТМО ур. 2 | `"Московская область"`                |
| `locality`     | string | нет          | Населённый пункт                | ОКТМО ур. 4   | `"Москва"`                            |
| `street`       | string | нет          | Улица                           | ГАР ур. 8     | `"Пролетарский проспект"`             |
| `house`        | string | нет          | Номер дома                      | ГАР ур. 10    | `"19"`                                |
| `housing`      | string | нет          | Корпус                          | ГАР ур. 10    | `"1"`                                 |
| `building`     | string | нет          | Строение                        | ГАР ур. 10    | `"1"`                                 |
| `apartment`    | string | нет          | Квартира                        | ГАР ур. 11    | `"2"`                                 |
| `comment`      | string | нет          | Комментарий к адресу            | —             | `"Подъезд 3, домофон 15"`             |
| `full_address` | string | нет          | Полный адрес строкой            | —             | `"Москва, Пролетарский проспект, 19"` |
| `postal_code`  | string | нет          | Почтовый индекс                 | —             | `"123182"`                            |

> **Все поля необязательные.** `LocationDetails` — информационный объект, заполняемый геокодером. API не валидирует отдельные поля адреса по справочникам.

## Уровни территориальной иерархии в API

### Уровень 0: Страна (`country`)

| Поле      | Значение   | Стандарт                       |
| --------- | ---------- | ------------------------------ |
| `country` | `"Россия"` | Строковое название, не ISO-код |

API работает с Россией и странами СНГ. Код страны не используется — только строковое название.

### Уровень 1: Регион (`region`)

Субъект Российской Федерации — верхний уровень территориального деления.

| Тип субъекта        | Пример в API           |
| ------------------- | ---------------------- |
| Область             | `"Московская область"` |
| Край                | `"Краснодарский край"` |
| Республика          | `"Татарстан"`          |
| Город фед. значения | `"Москва"`             |

> **Особенность:** Для городов федерального значения `region` и `locality` совпадают. Например: `"region": "Москва"`, `"locality": "Москва"`.

### Уровень 2: Подрегион (`subRegion`) — нестрогий

Поле `subRegion` **не соответствует строго** муниципальному району по ОКТМО. Оно заполняется геокодером Яндекса и может содержать:

- Название области для населённых пунктов внутри агломерации
- Название района для областных городов
- `null` или пустую строку

**Пример из API:**

```json
{
  "region": "Москва",
  "subRegion": "Московская область",
  "locality": "Москва"
}
```

> Здесь `subRegion: "Московская область"` — это артефакт геокодера, а не уровень деления.

### Уровень 3: Населённый пункт (`locality` + `geo_id`)

Населённый пункт идентифицируется двумя способами:

| Идентификатор      | Описание                       | Пример                       |
| ------------------ | ------------------------------ | ---------------------------- |
| `locality`         | Строковое название             | `"Москва"`, `"Екатеринбург"` |
| `geo_id` / `geoId` | Числовой ID из GeoBase Яндекса | `213` (Москва), `2` (СПб)    |

**`geo_id`** — это **не ОКТМО и не ФИАС**. Это внутренний идентификатор Яндекса (система GeoBase), используемый во всех сервисах Яндекса (Карты, Погода, Маркет и т.д.).

**Получение `geo_id`:**

```
POST /api/b2b/platform/location/detect
{"location": "Москва"}

→ {"variants": [{"geo_id": 213, "address": "Москва"}]}
```

> **Один город — несколько `geo_id`:** крупные города могут возвращать несколько вариантов в зависимости от строки запроса.

**Использование `geo_id`:**

- Фильтрация пунктов выдачи (`POST /api/b2b/platform/pickup-points/list`)
- Определение зоны доставки

### Уровень 4: Улица (`street`)

Улица, проспект, переулок, бульвар, набережная.

В плоской строке адреса включается после города через запятую:

```
"Москва, Пролетарский проспект, 19"
```

### Уровень 5: Здание (`house` + `housing` + `building`)

| Поле в API | Описание   | Аналог в ФИАС (ГАР) |
| ---------- | ---------- | ------------------- |
| `house`    | Номер дома | AS_HOUSES: дом      |
| `housing`  | Корпус     | AS_HOUSES: корпус   |
| `building` | Строение   | AS_HOUSES: строение |

> **Отличие от Почты России:** У Почты России 6 полей для здания (`house`, `building`, `corpus`, `letter`, `slash`, `vladenie`). У Yandex — только 3 (`house`, `housing`, `building`). Литера, дробь и владение **не выделяются** отдельно — включаются в `house`.

### Уровень 6: Квартира (`apartment`)

Номер квартиры. **Важно:** квартира **не включается** в строку `address` / `full_address` — передаётся только в `details.apartment`.

## Координаты — ключевой элемент маршрутизации

В отличие от Почты России (маршрутизация по почтовому индексу), Yandex Delivery использует **координаты** как основной инструмент маршрутизации:

```json
{
  "custom_location": {
    "latitude": 55.666624,
    "longitude": 37.51573,
    "details": { ... }
  }
}
```

| Модель               | Координаты              | Где используется                          |
| -------------------- | ----------------------- | ----------------------------------------- |
| `CustomLocation`     | `latitude`, `longitude` | Назначение заказа (курьерская доставка)   |
| `GeoPoint`           | `latitude`, `longitude` | Координаты ПВЗ (`PickupStation.position`) |
| `CoordinateInterval` | `from`, `to`            | Фильтрация ПВЗ по координатам             |

**Цепочка обработки адреса:**

```
Строка адреса → Геокодер Яндекса → Координаты → Маршрутизация
     ↓                                              ↓
full_address                                   Зона доставки
     ↓                                         Расчёт стоимости
LocationDetails (разбор)                       Назначение курьера
```

## Пункты выдачи заказов (ПВЗ) как альтернатива адресу

Вместо адресной иерархии можно указать идентификатор пункта выдачи:

| Тип ПВЗ (`PickupStationType`) | Описание                 |
| ----------------------------- | ------------------------ |
| `pickup_point`                | Пункт выдачи заказов     |
| `terminal`                    | Постамат (parcel locker) |
| `warehouse`                   | Сортировочный центр      |

Модель `PickupStation` содержит полную территориальную привязку:

```json
{
  "id": "0198602de4a6749aba12e151bdf4caaa",
  "name": "Пункт выдачи заказов Яндекс Маркета",
  "type": "pickup_point",
  "position": { "latitude": 55.666624, "longitude": 37.51573 },
  "address": {
    "geoId": 213,
    "country": "Россия",
    "region": "Москва",
    "subRegion": "Московская область",
    "locality": "Москва",
    "street": "Пролетарский проспект",
    "house": "19",
    "housing": "1",
    "building": "1",
    "full_address": "Москва, Пролетарский проспект, 19",
    "postal_code": "123182"
  }
}
```

**Фильтрация ПВЗ** (`POST /api/b2b/platform/pickup-points/list`):

| Критерий              | Поле запроса                                 | Описание                                |
| --------------------- | -------------------------------------------- | --------------------------------------- |
| По населённому пункту | `geo_id`                                     | Числовой идентификатор GeoBase          |
| По координатам        | `latitude`, `longitude` (CoordinateInterval) | Прямоугольная область                   |
| По ID                 | `pickup_point_ids`                           | Массив UUID                             |
| По типу               | `type`                                       | `pickup_point`, `terminal`, `warehouse` |
| По оператору          | `operator_ids`                               | `market_l4g`, `5post` и т.д.            |

## Сравнение полей адреса: Yandex Delivery vs. Почта России

| Уровень               | Yandex Delivery (`LocationDetails`) | Почта России (OrderV1, получатель) | Почта России (NormalizedAddress) |
| --------------------- | ----------------------------------- | ---------------------------------- | -------------------------------- |
| Страна                | `country`                           | — (подразумевается)                | —                                |
| Регион                | `region`                            | `region-to`                        | `region`                         |
| Район                 | `subRegion` (нестрого)              | `area-to`                          | `area`                           |
| Населённый пункт      | `locality`                          | `place-to`                         | `place`                          |
| ID населённого пункта | `geoId` (GeoBase)                   | — (нет)                            | — (нет)                          |
| Микрорайон            | — (нет)                             | `location-to`                      | `location`                       |
| Улица                 | `street`                            | `street-to`                        | `street`                         |
| Дом                   | `house`                             | `house-to`                         | `house`                          |
| Корпус                | `housing`                           | `corpus-to`                        | `corpus`                         |
| Строение              | `building`                          | `building-to`                      | `building`                       |
| Литера                | — (включена в `house`)              | `letter-to`                        | `letter`                         |
| Дробь                 | — (включена в `house`)              | `slash-to`                         | `slash`                          |
| Владение              | — (нет)                             | `vladenie-to`                      | —                                |
| Квартира              | `apartment`                         | `room-to`                          | `room`                           |
| Офис                  | — (нет)                             | `office-to`                        | —                                |
| Гостиница             | — (нет)                             | `hotel-to`                         | `hotel`                          |
| Полный адрес          | `full_address`                      | `raw-address`                      | `original-address`               |
| Почтовый индекс       | `postal_code`                       | `index-to`                         | `index`                          |
| Координаты            | `latitude` + `longitude`            | — (нет)                            | — (нет)                          |
| Комментарий           | `comment`                           | — (нет в адресе)                   | —                                |
| Тип адреса            | — (нет)                             | `address-type-to`                  | `address-type`                   |

**Ключевые различия:**

- Yandex: **11 полей** в `LocationDetails`, все необязательные, + координаты + `geo_id`
- Почта России: **16+ полей** адреса получателя, `region-to` и `place-to` обязательны, координат нет
- Yandex **не разделяет** литеру, дробь, владение — всё в `house`
- Yandex **не поддерживает** микрорайон, офис, гостиницу как отдельные поля
- Yandex **добавляет** координаты, `geo_id`, `comment`, `full_address` — у Почты России их нет

## Особые случаи

| Ситуация                   | Как обрабатывается в API                                                                                                                           |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| Москва, Питер, Севастополь | `region` = `locality` (например: `"region": "Москва"`, `"locality": "Москва"`). `subRegion` может быть `"Московская область"` — артефакт геокодера |
| Доставка в ПВЗ             | Адрес не нужен — используется `platform_station_id` (UUID)                                                                                         |
| Постамат                   | `type: "terminal"`, адрес — в поле `address` модели `PickupStation`                                                                                |
| Тестовая среда             | Только московские адреса (BR-027)                                                                                                                  |
| Квартира                   | **Не включается** в `address`/`full_address`. Передаётся отдельно в `details.apartment`                                                            |
| Домофон, подъезд, этаж     | Передаются в `comment`, не в полях адреса                                                                                                          |
| Международная доставка     | API ориентирован на внутреннюю доставку. `country` — строковое название                                                                            |

## Идентификаторы и классификаторы

| Идентификатор            | Система         | Описание                         | Пример                                   |
| ------------------------ | --------------- | -------------------------------- | ---------------------------------------- |
| `geo_id` / `geoId`       | GeoBase Яндекса | Внутренний ID населённого пункта | Москва = `213`, СПб = `2`                |
| `platform_station_id`    | Yandex Delivery | UUID пункта выдачи / склада      | `"e1139f6d-e34f-47a9-a55f-31f032a861a6"` |
| `operator_station_id`    | Оператор ПВЗ    | ID в системе оператора           | `"10035218565"`                          |
| `postal_code`            | Почта России    | 6-значный индекс (информационно) | `"123182"`                               |
| `latitude` + `longitude` | WGS-84          | Координаты точки доставки        | `55.666624`, `37.51573`                  |

> **`geo_id` ≠ ОКТМО ≠ ФИАС.** Это собственный идентификатор Яндекса. Для получения `geo_id` используется эндпоинт `POST /api/b2b/platform/location/detect`.

## Связь с государственными классификаторами

| Классификатор           | Как используется в API                                                                                     |
| ----------------------- | ---------------------------------------------------------------------------------------------------------- |
| **ОКТМО** (ОК 033-2013) | **Не используется.** Территориальная привязка — через `geo_id` и координаты                                |
| **ФИАС/ГАР**            | **Не используется для валидации.** Адрес геокодируется, а не сверяется со справочником                     |
| **КЛАДР**               | **Не используется.** Нет аналога `NOT_VALIDATED_HAS_NO_KLADR_RECORD`                                       |
| **ISO 3166-2:RU**       | **Не используется.** Регион — строковое название, не ISO-код                                               |
| **GeoBase Яндекса**     | **Основная система.** `geo_id` — внутренний идентификатор населённого пункта                               |
| **Почтовый индекс**     | **Информационное поле** `postal_code`. Не используется для маршрутизации — маршрут строится по координатам |

## API-эндпоинты, связанные с территориальной иерархией

| Эндпоинт                               | Метод    | Описание             | Территориальный аспект                             |
| -------------------------------------- | -------- | -------------------- | -------------------------------------------------- |
| `/api/b2b/platform/location/detect`    | POST     | Определение `geo_id` | Вход: строка адреса → Выход: `geo_id` + адрес      |
| `/api/b2b/platform/pickup-points/list` | POST     | Список ПВЗ           | Фильтрация по `geo_id`, координатам, типу          |
| `/api/b2b/platform/pricing-calculator` | POST     | Расчёт стоимости     | Вход: `address` строкой или `platform_station_id`  |
| `/api/b2b/platform/offers/info`        | GET/POST | Интервалы доставки   | Вход: `full_address` или `self_pickup_id`          |
| `/api/b2b/platform/offers/create`      | POST     | Создание заказа      | Полный адрес: `CustomLocation` + `LocationDetails` |

## Ошибки, связанные с адресами

| Код | Текст ошибки                                               | Причина                                                |
| --- | ---------------------------------------------------------- | ------------------------------------------------------ |
| 400 | `Cannot parse destination info`                            | Неверный формат адреса. Требуется: «Город, улица, дом» |
| 400 | `Cant get station id for point`                            | Невалидный `platform_station_id`                       |
| 400 | `Сant calc routes because destination station is disabled` | ПВЗ деактивирован                                      |
| 404 | `No delivery options`                                      | Нет доступных вариантов доставки для указанного адреса |
