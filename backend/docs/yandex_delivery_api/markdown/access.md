## Боевой доступ

Все запросы к API Яндекс Доставки в другой день выполняются к хосту:

```
https://b2b-authproxy.taxi.yandex.net
```

Пример запроса к боевому хосту:

```
https://b2b-authproxy.taxi.yandex.net/api/b2b/platform/offers/create
```

### Аутентификация (боевая)

| Параметр | Значение |
|----------|----------|
| Bearer-токен | Получить в [личном кабинете](https://dostavka.yandex.ru/account), раздел **Интеграция** |
| Срок действия токена | Неограниченный |
| Условие инвалидации | При смене пароля от [личного кабинета](https://dostavka.yandex.ru/login/) токен перестаёт действовать и требует повторного получения |

Используйте Bearer-токен во всех запросах. Передавайте токен в заголовке `Authorization`:

```
Authorization: Bearer <OAuth-токен>
```

## Тестовый доступ

> **Примечание:** В тестовой среде поддерживается обработка адресов только по г. Москва (МСК). Данное ограничение распространяется как на курьерскую доставку, так и на доставку в пункты выдачи заказов (ПВЗ) и почтовые отделения (ПСТ).

Все запросы к API Яндекс Доставки в другой день выполняются к хосту:

```
https://b2b.taxi.tst.yandex.net/
```

Пример запроса к тестовому хосту:

```
https://b2b.taxi.tst.yandex.net/api/b2b/platform/offers/create
```

## Получение OAuth-токена

| Шаг | Действие |
|-----|----------|
| 1 | Войдите в [личный кабинет](https://dostavka.yandex.ru/login/) с помощью выданного вам логина и пароля |
| 2 | Перейдите на вкладку **[Интеграция](https://dostavka.yandex.ru/account2/integration)** |
| 3 | В разделе **Активируйте интеграцию** нажмите **Получить токен** |
| 4 | Ваш OAuth-токен отображается в личном кабинете Яндекс Доставки на вкладке **Профиль компании** → **Токен для API** |
| 5 | Создайте заявку и отслеживайте состояние заказа с помощью API в соответствии с порядком обработки заявки |
