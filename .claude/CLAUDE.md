# Архитектурные стандарты проекта (FastAPI High-Load)

## Базовая архитектура (Modular Monolith & Hexagonal)
- Код разделен на независимые модули (Bounded Contexts) внутри `src/modules/`.
- Модули не могут напрямую импортировать логику друг из друга. Взаимодействие происходит только через публичные интерфейсы (Integration Events) или внутренний Mediator.
- Строго соблюдается направление зависимостей: `infrastructure` -> `application` -> `domain`.

## Domain-Driven Design (DDD) & Ubiquitous Language
- Слой `domain` содержит только чистый Python. ЗАПРЕЩЕНЫ импорты FastAPI, Pydantic, SQLAlchemy или любых внешних библиотек.
- Сущности (Entities) и объекты-значения (Value Objects) должны инкапсулировать бизнес-логику. Анемичные модели запрещены.
- Используйте строгий Ubiquitous Language в названиях: классы, переменные и методы должны называться исключительно терминами предметной области.

## CQRS (Command Query Responsibility Segregation)
- Разделяйте операции на уровне слоя `application`.
- **Commands**: изменяют состояние. Используют паттерн Repository для доступа к домену. Возвращают только статус успеха/ошибки или ID созданной сущности.
- **Queries**: только чтение. Не используют репозитории или доменные модели. Выполняют сырые оптимизированные запросы (SQLAlchemy/SQL) напрямую к базе данных и возвращают плоские Pydantic DTO (Read Models).

## FastAPI (Presentation Layer)
- Роутеры (Endpoints) располагаются строго в слое `presentation`.
- FastAPI роутер — это исключительно фасад: он принимает запрос, валидирует входные данные, вызывает Command/Query Handler из `application` слоя и возвращает ответ.
- Наличие бизнес-логики в FastAPI роутерах строго запрещено.

## Infrastructure Layer
- Вся работа с БД (SQLAlchemy/Alembic), Redis и внешними API находится здесь.
- Инфраструктура реализует интерфейсы (Protocol / ABC), определенные в слое `application` или `domain`. Никаких протечек инфраструктурных деталей на уровни выше.
