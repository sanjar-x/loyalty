# Plan 07-03: Schema Constraint Audit & Soft-Delete Filter Verification

<objective>
Verify that all FK, unique, and check constraints defined in migrations are enforced at the database level (not just application validation). Systematically audit every ProductRepository method that reads data to confirm soft-deleted records (deleted_at IS NOT NULL) are correctly excluded. This validates that the schema is the ultimate guard against data corruption and that no query leaks soft-deleted records.
</objective>

<must_haves>
- FK constraints reject orphaned records (product without brand, SKU without product, etc.)
- Unique constraints reject duplicate slugs, codes, and composite keys at DB level
- Partial unique indexes (products.slug WHERE deleted_at IS NULL) allow duplicate slugs on soft-deleted records
- Check constraint (search_weight BETWEEN 1 AND 10) rejects out-of-range values at DB level
- ON DELETE CASCADE correctly removes child rows when parent is deleted
- ON DELETE RESTRICT correctly prevents parent deletion when children exist
- Every ProductRepository read method filters out soft-deleted Products
- get_with_variants filters out soft-deleted Variants and SKUs within an active Product
- check_slug_exists and check_sku_code_exists exclude soft-deleted records
- BrandRepository.has_products and CategoryRepository.has_products exclude soft-deleted products
</must_haves>

4. `test_sku_requires_valid_variant_id` -- Insert SKU with variant_id=uuid.uuid4(). Assert IntegrityError.

5. `test_sku_requires_valid_currency` -- Insert SKU with currency="XXX" (not in currencies table). Assert IntegrityError on the FK to currencies.code.

6. `test_attribute_value_requires_valid_attribute_id` -- Insert AttributeValue with attribute_id=uuid.uuid4(). Assert IntegrityError.

7. `test_template_binding_requires_valid_template_id` -- Insert TemplateAttributeBinding with template_id=uuid.uuid4(). Assert IntegrityError.

8. `test_template_binding_requires_valid_attribute_id` -- Insert TemplateAttributeBinding with attribute_id=uuid.uuid4(). Assert IntegrityError.

For each test:
- Use direct ORM model construction (not domain entity factories -- we want to bypass domain validation)
- session.add(orm_model) then await session.flush()
- Wrap in pytest.raises(IntegrityError) or try/except with rollback
- Since nested transaction will auto-rollback, use session.begin_nested() before the flush to isolate the error

Pattern:
```python
async with db_session.begin_nested():
    orm = OrmProduct(id=uuid.uuid7(), brand_id=uuid.uuid4(), ...)
    db_session.add(orm)
    with pytest.raises(IntegrityError):
        await db_session.flush()
```
