# Plan 07-02: Attribute, Template & Supporting Repository CRUD Integration Tests

<objective>
Prove that all non-Product catalog repositories (Attribute, AttributeValue, AttributeTemplate, AttributeGroup, TemplateAttributeBinding, MediaAsset, ProductAttributeValue) correctly perform CRUD operations against real PostgreSQL with accurate ORM-to-domain mapping. Verify BehaviorFlags VO decomposition on Attribute, JSONB i18n roundtrips, scoped uniqueness checks, and specialized query methods (tree operations on Category already covered by existing tests).
</objective>

<must_haves>
- AttributeRepository roundtrips all fields including BehaviorFlags VO decomposition (5 columns -> 1 VO)
- AttributeValueRepository roundtrips JSONB (value_i18n, meta_data) and ARRAY (search_aliases) fields
- AttributeTemplateRepository roundtrips code and JSONB i18n fields
- AttributeGroupRepository roundtrips code and JSONB i18n fields
- TemplateAttributeBindingRepository roundtrips requirement_level enum and filter_settings JSONB
- MediaAssetRepository roundtrips all fields including enum (MediaType, MediaRole) mapping
- ProductAttributeValueRepository roundtrips FK triple (product_id, attribute_id, attribute_value_id)
- All repository get_many/check/has methods verified
</must_haves>
