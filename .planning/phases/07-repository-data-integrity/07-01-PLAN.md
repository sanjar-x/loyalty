# Plan 07-01: Product Repository 3-Level Roundtrip & ORM Mapping Fidelity

<objective>
Prove the ProductRepository's Data Mapper correctly roundtrips the full Product aggregate (Product -> ProductVariant -> SKU -> SKUAttributeValueLink) through real PostgreSQL. Verify Money VO decomposition, JSONB i18n fields, nullable fields, datetime preservation, enum mapping, and N+1 query detection on eager loading. This is the highest-risk and most complex repository in the catalog module.
</objective>

<must_haves>
- Product with variants and SKUs survives full create-read roundtrip with all fields intact
- Money VO (price, compare_at_price) correctly decomposes to/from integer + currency columns
- JSONB fields (title_i18n, description_i18n, name_i18n) roundtrip without data loss
- ARRAY fields (tags) roundtrip correctly
- Nullable fields (source_url, supplier_id, compare_at_price, published_at) preserve None
- UUID fields (uuid7) survive roundtrip without type coercion
- Enum fields (ProductStatus) map correctly between domain StrEnum and DB
- SKU variant_attributes (link table) roundtrip correctly
- N+1 detection verifies bounded query count on get_with_variants
- ProductRepository.update() with _sync_variants reconciles additions, updates, removals
</must_haves>
