<objective>
Fix ProductCreateRequest to make descriptionI18n truly optional (BKND-01) and add countryOfOrigin field with router wiring (BKND-02). Also fix the identical description_i18n bug in AttributeCreateRequest and AttributeTemplateCreateRequest (discretionary, same one-line fix).

Purpose: These are the two backend schema bugs blocking product creation from the admin panel. Without BKND-01, any product creation request that omits descriptionI18n gets a 422 validation error. Without BKND-02, countryOfOrigin cannot be set at creation time.

Output: Fixed schema/command/router files, 6 new e2e tests proving both behaviors.
</objective>

async def create_product(client, brand_id, category_id, *, slug=None, title_i18n=None) -> dict:
    slug = slug or f"prod-{uuid.uuid4().hex[:8]}"
    title_i18n = title_i18n or {"ru": "Товар", "en": "Product"}
    payload = {
        "titleI18n": title_i18n,
        "slug": slug,
        "brandId": str(brand_id),
        "primaryCategoryId": str(category_id),
    }
    resp = await client.post("/api/v1/catalog/products", json=payload)
    assert resp.status_code == 201
    return resp.json()
