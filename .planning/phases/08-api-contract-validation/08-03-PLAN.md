  **test_storefront.py** (API-02) -- Test all 4 storefront endpoints:

  Setup: Each test needs a category with an assigned template that has attribute bindings. The storefront endpoints return attribute metadata based on category templates.

  Setup flow (reusable within each test or as a module-level helper):
  1. Create attribute template
  2. Create attribute (is_filterable=True, is_visible_on_card=True, is_comparable=True, is_dictionary=True)
  3. Add attribute values to the attribute
  4. Bind attribute to template (requirement_level="required")
  5. Create category with template_id pointing to the template

  Tests to write:
  - test_get_filterable_attributes_success: GET /storefront/categories/{cid}/filters -> 200. Response shape: categoryId, attributes array. Each attribute has: attributeId, code, slug, nameI18n, dataType, uiType, isDictionary, selectionMode, values array, sortOrder. The endpoint is PUBLIC (no auth needed) -- use `async_client` directly.
  - test_get_filterable_attributes_with_lang_projection: GET /storefront/categories/{cid}/filters?lang=ru -> 200. Attribute items should have a `name` field (projected from nameI18n).
  - test_get_card_attributes_success: GET /storefront/categories/{cid}/card-attributes -> 200. Response shape: categoryId, groups array. Each group has: groupId, groupCode, groupNameI18n, groupSortOrder, attributes array.
  - test_get_comparison_attributes_success: GET /storefront/categories/{cid}/comparison-attributes -> 200. Response shape: categoryId, attributes array.
  - test_get_form_attributes_success: GET /storefront/categories/{cid}/form-attributes -> 200. This endpoint requires catalog:manage (use admin_client). Response shape: categoryId, groups array with full attribute details.
  - test_storefront_filters_no_template_returns_empty: category without template -> 200 with empty attributes list
  - test_storefront_form_attributes_requires_auth: anonymous request to form-attributes -> 401 (this endpoint requires catalog:manage unlike the other 3 public ones)

  IMPORTANT: The public storefront endpoints (filters, card-attributes, comparison-attributes) do NOT require authentication. Use `async_client` (raw client without auth header). Only form-attributes requires `catalog:manage`.

  **test_lifecycle.py** (API-04) -- One comprehensive test proving the entire catalog stack works end-to-end:

  ```python
  async def test_full_product_lifecycle(admin_client, db_session):
      """Full product lifecycle: create -> enrich -> review -> publish.

      This test proves the entire catalog API stack works end-to-end through
      the HTTP layer by executing the complete product creation flow documented
      in docs/api/product-creation-flow.md.
      """
      # Step 1: Create prerequisites
      brand = await create_brand(admin_client)
      template = await create_attribute_template(admin_client)
      attr = await create_attribute(admin_client, level="variant", is_dictionary=True, is_filterable=True)
      val1 = await create_attribute_value(admin_client, attr["id"], code="red", slug="red", value_i18n={"ru": "Красный", "en": "Red"})
      val2 = await create_attribute_value(admin_client, attr["id"], code="blue", slug="blue", value_i18n={"ru": "Синий", "en": "Blue"})
      binding = await bind_attribute_to_template(admin_client, template["id"], attr["id"])
      category = await create_category(admin_client, template_id=template["id"])

      # Step 2: Create product (DRAFT status)
      product_resp = await admin_client.post("/api/v1/catalog/products", json={
          "titleI18n": {"ru": "Тестовый товар", "en": "Test Product"},
          "slug": f"lifecycle-{uuid4().hex[:8]}",
          "brandId": str(brand["id"]),
          "primaryCategoryId": str(category["id"])
      })
      assert product_resp.status_code == 201
      product = product_resp.json()
      product_id = product["id"]
      variant_id = product["defaultVariantId"]

      # Step 3: Add SKU with price
      sku_resp = await admin_client.post(
          f"/api/v1/catalog/products/{product_id}/variants/{variant_id}/skus",
          json={
              "skuCode": f"LIFE-{uuid4().hex[:6]}",
              "priceAmount": 9990,
              "priceCurrency": "RUB",
              "isActive": True,
              "variantAttributes": [
                  {"attributeId": str(attr["id"]), "attributeValueId": str(val1["id"])}
              ]
          }
      )
      assert sku_resp.status_code == 201

      # Step 4: Add external media asset (bypasses ImageBackend)
      media_resp = await admin_client.post(
          f"/api/v1/catalog/products/{product_id}/media",
          json={
              "isExternal": True,
              "url": "https://example.com/lifecycle-photo.jpg",
              "mediaType": "image",
              "role": "main",
              "sortOrder": 0
          }
      )
      assert media_resp.status_code == 201

      # Step 5: Transition through FSM states
      # DRAFT -> ENRICHING
      resp = await admin_client.patch(
          f"/api/v1/catalog/products/{product_id}/status",
          json={"status": "enriching"}
      )
      assert resp.status_code == 200
      assert resp.json()["status"] == "enriching"

      # ENRICHING -> READY_FOR_REVIEW
      resp = await admin_client.patch(
          f"/api/v1/catalog/products/{product_id}/status",
          json={"status": "ready_for_review"}
      )
      assert resp.status_code == 200
      assert resp.json()["status"] == "ready_for_review"

      # READY_FOR_REVIEW -> PUBLISHED
      resp = await admin_client.patch(
          f"/api/v1/catalog/products/{product_id}/status",
          json={"status": "published"}
      )
      assert resp.status_code == 200
      product_data = resp.json()
      assert product_data["status"] == "published"
      assert product_data["publishedAt"] is not None

      # Step 6: Verify product detail has all nested data
      detail_resp = await admin_client.get(f"/api/v1/catalog/products/{product_id}")
      assert detail_resp.status_code == 200
      detail = detail_resp.json()
      assert detail["status"] == "published"
      assert len(detail["variants"]) >= 1
      assert any(len(v["skus"]) > 0 for v in detail["variants"])

      # Step 7: Verify product appears in list
      list_resp = await admin_client.get("/api/v1/catalog/products", params={"status": "published"})
      assert list_resp.status_code == 200
      published_ids = [p["id"] for p in list_resp.json()["items"]]
      assert product_id in published_ids
  ```

  The test must be self-contained -- create everything it needs within the test function. Use conftest helpers for prerequisites. Import from conftest: create_brand, create_category, create_attribute, create_attribute_value, create_attribute_template, bind_attribute_to_template.

  **test_pagination.py** (API-05) -- Test pagination behavior on brands and products list endpoints:

  Tests to write:

  **Brands pagination:**
  - test_brands_pagination_default_params: GET /catalog/brands (no params) -> 200, response has offset=0, limit=50, total >= 0, hasNext is bool
  - test_brands_pagination_custom_offset_limit: create 3 brands, GET with offset=1&limit=2 -> items array has at most 2 items, offset=1, limit=2
  - test_brands_pagination_offset_beyond_total: create 1 brand, GET with offset=1000 -> 200 with items=[], total >= 1 (total reflects real count, items is empty because offset is past it)
  - test_brands_pagination_has_next_true: create 3 brands, GET with limit=1 -> hasNext=true (more items exist)
  - test_brands_pagination_has_next_false: create 1 brand, GET with limit=100 -> hasNext=false

  **Products pagination:**
  - test_products_pagination_default_params: GET /catalog/products (no explicit offset/limit) -> 200, response has offset=0, total, hasNext
  - test_products_pagination_custom_limit: create 3 products, GET with limit=2 -> items array length <= 2

  **Edge cases:**
  - test_pagination_empty_result: GET /catalog/brands with a very high offset -> 200 with items=[] and total=0 (or total >= 0 if brands exist)
  - test_pagination_limit_max_boundary: GET /catalog/brands with limit=200 (max allowed) -> 200 (should not error)
  - test_pagination_limit_exceeds_max_returns_422: GET /catalog/brands with limit=201 -> 422 (exceeds le=200 validation)

  Key assertions for every pagination test:
  1. `items` is a list
  2. `total` is an integer >= 0
  3. `offset` matches the requested offset
  4. `limit` matches the requested limit
  5. `hasNext` is a boolean
  6. `len(items) <= limit`
  7. When `hasNext` is true: `offset + len(items) < total`
