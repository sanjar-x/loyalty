    **File 1: `backend/tests/utils/query_counter.py`** (per D-09)

    Build `assert_query_count` as an ASYNC context manager (not sync) using SQLAlchemy's `after_cursor_execute` event. The async approach is required because accessing the underlying sync connection from an async session requires `await`.

    ```python
    from __future__ import annotations
    from contextlib import asynccontextmanager
    from sqlalchemy import event
    from sqlalchemy.ext.asyncio import AsyncSession

    @asynccontextmanager
    async def assert_query_count(session: AsyncSession, expected: int, *, label: str = ""):
        """Async context manager that counts SQL queries and asserts exact count.

        Hooks into SQLAlchemy's after_cursor_execute event on the underlying
        sync connection. Filters out SAVEPOINT, RELEASE SAVEPOINT, and
        ROLLBACK TO SAVEPOINT statements since those are artifacts of the
        nested-transaction test isolation pattern.

        Args:
            session: The async session to monitor.
            expected: Expected number of application-level SQL queries.
            label: Optional label for the assertion error message.

        Usage:
            async with assert_query_count(db_session, expected=1, label="list_brands"):
                result = await brand_repo.list_all()
        """
    ```

    CRITICAL implementation details per research pitfall #6:
    1. Get the sync connection: `sync_conn = await session.connection()` returns an `AsyncConnection`
    2. Get the raw sync connection: `raw_conn = sync_conn.sync_connection` (the underlying sync Connection object)
    3. Attach `after_cursor_execute` event listener to `raw_conn`
    4. In the listener callback, increment counter but SKIP statements containing:
       - `SAVEPOINT` (covers both SAVEPOINT and RELEASE SAVEPOINT)
       - `ROLLBACK TO SAVEPOINT`
       These are test isolation artifacts, not application queries.
    5. On exit from context manager, remove the listener and assert `count == expected`

    The listener function signature must match SQLAlchemy's `after_cursor_execute` event:
    ```python
    def _count_queries(conn, cursor, statement, parameters, context, executemany):
        # Skip SAVEPOINT-related statements
        stmt_upper = statement.strip().upper()
        if stmt_upper.startswith(("SAVEPOINT", "RELEASE SAVEPOINT", "ROLLBACK TO SAVEPOINT")):
            return
        counter["count"] += 1
    ```

    Error message format: `f"Expected {expected} queries{label_str}, got {counter['count']}"`
