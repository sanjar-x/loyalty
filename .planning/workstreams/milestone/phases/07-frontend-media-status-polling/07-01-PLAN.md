<objective>
Add media processing status polling to the admin frontend so that after upload confirm (202 Accepted), the browser waits for image processing to reach COMPLETED before proceeding. This closes audit issue #9 (MAJOR): without polling, url and variants fields are still empty when the product references them.

Purpose: Ensure media is fully processed (url and variants populated) before the upload step is considered done.
Output: BFF proxy route for media metadata, pollMediaStatus service function, wired into useSubmitProduct hook.
</objective>

   - Exponential backoff intervals: [500, 1000, 2000] ms, then repeats 2000ms

   - CRITICAL: Compare status against UPPERCASE strings ('COMPLETED', 'FAILED') -- image_backend StorageStatus enum values are UPPERCASE
