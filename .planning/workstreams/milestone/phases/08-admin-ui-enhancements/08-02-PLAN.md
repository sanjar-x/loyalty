1. Create `frontend/admin/src/app/admin/products/[productId]/page.jsx`:

   'use client' directive at top.

   Imports:
   - `{ useState, useEffect, useCallback }` from 'react'
   - `{ useParams }` from 'next/navigation'
   - `Link` from 'next/link'
   - `{ getProduct, getProductCompleteness, changeProductStatus }` from '@/services/products'

   - `{ CompletenessPanel }` from '@/components/admin/products/CompletenessPanel'
   - `{ StatusTransitionBar }` from '@/components/admin/products/StatusTransitionBar'
   - `styles` from './page.module.css'

   Component: `export default function ProductDetailPage()`

   State:
   - `product` (null initially) — full product object from API, includes `version`
   - `completeness` (null initially) — completeness response object
   - `loading` (true initially) — page loading state
   - `transitioning` (false initially) — status change in progress
   - `error` (null initially) — error message string

   Data fetching:
   - Extract `productId` from `useParams()`
   - `useEffect` on mount: call `loadProduct()` which fetches both product and completeness in parallel:
     ```javascript
     const loadProduct = useCallback(async () => {
       setLoading(true);
       setError(null);
       try {
         const [prod, comp] = await Promise.all([
           getProduct(productId),
           getProductCompleteness(productId),
         ]);
         setProduct(prod);
         setCompleteness(comp);
       } catch (err) {
         setError(err.message ?? 'Не удалось загрузить продукт');
       } finally {
         setLoading(false);
       }
     }, [productId]);
     ```
   - CRITICAL for UI-03: `product.version` is automatically tracked because the entire product object is stored in state. Any subsequent PATCH call can read `product.version`.

   Status transition handler:
   ```javascript
   const handleTransition = useCallback(async (targetStatus) => {
     if (!product) return;
     setTransitioning(true);
     setError(null);
     try {
       const updated = await changeProductStatus(product.id, targetStatus);
       // updated is the full ProductResponse with new version
       setProduct(updated);
       // Re-fetch completeness since status change might affect it
       const comp = await getProductCompleteness(product.id);
       setCompleteness(comp);
     } catch (err) {
       setError(err.message ?? 'Не удалось изменить статус');
     } finally {
       setTransitioning(false);
     }
   }, [product]);
   ```
   CRITICAL: After `changeProductStatus` returns, `updated.version` is the new version. By setting `setProduct(updated)`, version is automatically refreshed for any future PATCH calls.

   Rendering:
   - If `loading` and no product: show loading skeleton (simple div with "Загрузка..." text)
   - If `error` and no product: show error message with a "Попробовать снова" retry button that calls `loadProduct()`
   - If `product`:
     a) Header section:
        - Back link: `<Link href="/admin/products">` with left arrow and "Товары" text
        - Product title: `i18n(product.titleI18N)` in h1
        - Current status badge: `PRODUCT_STATUS_LABELS[product.status]` in a colored pill
        - Product slug shown as secondary text
        - Version indicator: small text "v{product.version}" for transparency

     b) Status transition section:
        - `<StatusTransitionBar status={product.status} loading={transitioning} onTransition={handleTransition} />`

     c) Error banner (if error is set while product exists):
        - Red banner with error text and dismiss button

     d) Content area (two-column layout on desktop, single column on mobile):
        - Left column (main): Product info card with:
          - Title (i18n)
          - Description (i18n, if present)
          - Brand ID, Category ID, Country of Origin (if present)
          - Created at, Updated at (formatted)
          - Status
        - Right column (sidebar): `<CompletenessPanel completeness={completeness} />`

   Layout: Use CSS Grid — `grid-template-columns: 1fr 360px` on large screens, single column on mobile.
