<objective>
Fix ProductCreateRequest to make descriptionI18n truly optional (BKND-01) and add countryOfOrigin field with router wiring (BKND-02). Also fix the identical description_i18n default bug in AttributeCreateRequest and AttributeTemplateCreateRequest (same one-line pattern, prevents 422 on optional description omission).

Purpose: These are the two backend schema bugs blocking product creation from the admin panel. Without BKND-01, any product creation request that omits descriptionI18n gets a 422 because the `default_factory=dict` produces `{}` which triggers `_validate_i18n_keys` requiring `{ru, en}` keys. Without BKND-02, countryOfOrigin cannot be set at creation time even though the command and domain already support it.

Output: Fixed schema/command/router files, 9 e2e tests proving all behaviors including attribute schema fixes and domain None-to-{} conversion path.
</objective>
