**Step 1: Add `buildI18nPayload` to `frontend/admin/src/lib/utils.js`**

Add the following function after the existing `i18n()` function (after line 51), before `copyToClipboard`:

```javascript
export function buildI18nPayload(ru, en) {
  return { ru, en: en || ru };
}
```

This helper guarantees both `ru` and `en` keys are always present. When `en` is falsy (empty string, undefined, null), it falls back to the `ru` value. Per D-01: "copy ru value to en before sending to backend."

**Step 3: Fix CategoriesPage page.jsx -- handleEdit extracting name from i18n dict**

Fix the `handleEdit` function (lines 39-49). Change from:

```javascript
function handleEdit(category) {
  setModal({
    mode: 'edit',

      name: category.name,
      slug: category.slug,
      sortOrder: category.sortOrder,
    },
```

```javascript
}
```

This extracts the Russian name string from the `nameI18N` dict and passes it as a plain string to CategoryModal, where the user can edit it. On save, CategoryModal wraps it back into `nameI18N` dict via `buildI18nPayload`.
