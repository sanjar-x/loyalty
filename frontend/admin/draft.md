3. План реализации

Шаг 1: Refactor state — единый ProductFormState

- Компоненты: useProductForm hook (новый)
- API: нет
- Данные: Собрать все поля в один state object:
  { categoryId, brandId, title, slug, selectedProductAttrs, selectedVariantAttrs,
  images, sourceUrl, supplierId, deliveryMode, priceAmount, compareAtPrice,
  variablePricing, perSizePrices }
- Зависимости: нет — фундамент для всех шагов

Шаг 2: Подключить BrandSelect к API

- Компоненты: BrandSelect.jsx (уже есть, refactor)
- API: GET /catalog/brands → list
- Данные: brandId в form state
- Текущее состояние: частично работает, нужно проверить API route

Шаг 3: Подключить DynamicAttributes к form state с level-разделением

- Компоненты: DynamicAttributes.jsx (уже есть, refactor)
- API: GET .../form-attributes (уже подключен)
- Изменения: Разделить атрибуты на product и variant в UI. Product attrs = single select, variant attrs = multi-select
- Зависимости: Шаг 1

Шаг 4: Подключить SizeSelect к variant-level атрибутам из API

- Компоненты: SizeSelect.jsx (refactor — убрать hardcoded sizes, использовать данные из form-attributes)
- API: данные уже загружены в Step 3
- Данные: selectedVariantAttrs в form state
- Зависимости: Шаг 3

Шаг 5: Подключить DeliverySection + supplier select к API

- Компоненты: DeliverySection.jsx (refactor)
- API: GET /suppliers → list
- Данные: supplierId, sourceUrl, deliveryMode в form state
- Зависимости: Шаг 1

Шаг 6: Подключить ImagesSection к media upload API

- Компоненты: ImagesSection.jsx (refactor)
- API: POST .../media/upload → presigned URL → PUT to S3 → POST .../confirm
- Данные: images[] с upload status tracking
- Зависимости: Шаг 1 (нужен productId для upload, но upload будет после submit)

Шаг 7: Секция "Цена" — подключить к SKU generation

- Компоненты: Price section (уже в ProductDetailsForm)
- API: priceAmount / compareAtPriceAmount идут в POST .../skus/generate
- Данные: priceAmount, compareAtPrice, variablePricing, perSizePrices
- Вариативная цена: Если toggle ON — после generate обновить каждый SKU через PATCH .../skus/{id}
- Зависимости: Шаг 4 (нужны выбранные sizes)

Шаг 8: Submit flow — цепочка API вызовов

- Компоненты: handleSubmit() функция, SubmitButton с loading state
- API chain:
  a. POST /catalog/products → { id, defaultVariantId }
  b. POST /products/{id}/attributes/bulk → product attrs
  c. POST .../skus/generate → variant attrs + price
  d. POST .../media/upload → for each image → S3 → confirm
  e. PATCH .../status → { status: "enriching" } (или "draft")
- Error handling: Каждый шаг может fail. Показать на каком шаге ошибка. Retry для media uploads.
- Зависимости: Шаги 1-7

Шаг 9: Preview sidebar

- Компоненты: ProductPreview.jsx (новый)
- Данные: Реактивно собирается из form state — название, первое фото, цена
- Зависимости: Шаги 1-7

Контекст

Есть план из 9 шагов по интеграции фронтенда с API. Это фундамент — все остальные шаги зависят от него. Сейчас state разбросан по компонентам, нужно
собрать в один hook.

Источники

- API контракты: @openapi.json
- Flow интеграции: @docs/product-attributes-integration.md
- UI: @create_product_page.png
- Текущие компоненты страницы: @src/app/(admin)/catalog/products/create/

Задача

Изучи стркутуру данных в API контракты и Flow интеграции и Создай hook useProductForm с единым state object:

Требования

1. Изучи существующие компоненты — найди где сейчас живёт state (useState, useForm, context) и собери всё в hook
2. Типы полей и naming — строго по openapi.json (camelCase)
3. Hook должен возвращать: state, setters/handlers, resetForm, isValid (базовая валидация обязательных полей)
4. Подключи hook к существующим компонентам — замени их локальный state на поля из hook
5. Не ломай текущий UI — компоненты должны работать как раньше, только source of truth меняется

Границы — НЕ делай

- Не подключай API вызовы — это Шаг 2+
- Не создавай submit flow — это Шаг 8
- Не меняй визуал компонентов
- Не добавляй новые UI элементы

Правила

- Перед изменением компонента покажи что именно заменяешь (было → стало)
- Если в компоненте нет state (чисто презентационный) — не трогай его
- Если naming в текущем коде расходится с openapi.json — приведи к openapi
